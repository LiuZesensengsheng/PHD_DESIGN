## Current Status

- Combat card-play and top-of-draw mainline are now queue-only.
- Active fallback allowlist remains at `0`.
- Direct execution helpers still exist in `EffectExecutor`, but they are now explicitly outside mainline ownership.
- The remaining direct helpers now use explicit direct-exec naming instead of fallback-shaped names.
- `reposition` no longer silently borrows the pointer target when card input is incomplete.
- Headless simulation now fails explicitly on missing enemies or missing card packs instead of silently patching configuration holes.
- Shared enemy damage commit no longer fabricates success if `commit_damage(...)` fails.
- Demo trait damage reactions no longer bypass the shared damage service with direct health writes on failure.
- Enemy-turn alias flows now share the same phase-machine ownership as the mainline and stop on step failures.
- Legacy enemy skill selection no longer hides cooldown/condition filtering bugs by picking a filtered-out candidate anyway.
- Active runtime white-pack data now formally owns the heart-demon curse cards instead of relying on a Python-side builder fallback.
- Direct `SELECTED_ENEMY` target resolution no longer silently hits the first enemy when input is incomplete.
- Thesis judgment now treats missing submission tier data as a state invariant failure instead of silently grading at `T2`.

## Next Step

1. Keep new combat effect families behind planner/action contracts by default.
2. Treat `QueueOnlyPlanningError` as a content/modeling signal, not as something to patch over with a new silent fallback.
3. Revisit whether any remaining simulation/runtime convenience fallback should become explicit configuration contracts too.
