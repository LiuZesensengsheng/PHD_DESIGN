## Current Status

- The trigger surface is no longer just a passive collector; it now has a real
  first consumer layer with explicit runtime ownership.
- Built-in campaign reactions are more centralized:
  - lifecycle machine still owns phase/step/window timing
  - task-area domain events still own factual board changes
  - trigger reactions now own stable reactive consumers
- This makes future trait/event work more AI-friendly because new reactions no
  longer need to choose between:
  - lifecycle hooks
  - raw `domain_events`
  - ad-hoc interrupt glue

  - forced events should move to a narrow long-term path
  - but should not widen into a generalized interrupt-platform rewrite yet
- Forced-event Slice 1 is now landed in working tree:
  - explicit model exists
  - pending queue ownership exists
  - presentation now lives behind a dedicated presenter seam
  - trigger reactions now only detect and enqueue forced events
  - active and pending forced-event states are now explicit
  - gate retry/order behavior is now deterministic
  - the next slice is the first extension guardrail
