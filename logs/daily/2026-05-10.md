## Boundary Notes

- STS1 campaign curve analysis is kept separate from DD1 stress, virtue, and
  affliction loops.
- The new campaign-curve artifacts are report-only documentation. They do not
  modify runtime campaign, design new cards, create hard gates, claim reviewed
  evidence, or change default `card_package_exam_v1` behavior.
- The new campaign curve profile fixtures are planning ratios and event-intent
  budgets, not copied STS1 map generation rules or runtime route generation.
- Campaign curve fit inside `card_package_exam_v1` is advisory only. It may surface
  risks such as `too_slow`, `too_narrow`, `recovery_window_collapse`,
  `elite_check_failure`, and `act2_transition_shock`, but it does not create a pass,
  fail, hard gate, or runtime campaign change.
- STS1 source material is treated as `source_mined_design_reference`, not project
  reviewed evidence.
- No LLM API was called.
- No complete card draft text was generated by runtime code.
- No runtime card data, formal cards, hard gates, reviewed evidence,
  default synthesis, learned behavior, or reranker behavior changed.
- Owner approval in the new manifest means approval for report-only intake and
  exam feedback only. It is not approval for card promotion or reviewed evidence.
