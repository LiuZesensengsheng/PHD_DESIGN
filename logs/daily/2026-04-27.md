## Candidate Long-Term Memory

- Gossip/BBS social simulation is currently an exploration-only model. It emits
  structured state payloads, not final forum prose.
- The model should stay sidecar-shaped until a campaign/narrative integration point is
  explicitly chosen.
- Debunked rumor memory should keep both the original claim and the corrected status;
  do not erase the old rumor as if it never existed.
- `misunderstanding` is a separate path from `rumor_seeded`: low malice and low
  correction access keep damage repairable, while repeated behavior after clarification
  should escalate.
- `new_card_package_seen` accepts only public structured package signals in this lab;
  it must not call or imply ownership by `cardanalysis` internals.
- `public_achievement` can strengthen public success memory and slow decay, but it
  must not certify exact package identity or turn into reward/narrative execution.
- `faction_event` can create alignment drift, but unknown player consent must remain
  explicit so faction identity does not overwrite actor agency.
- `npc_conflict` should remain separate from `rumor_seeded` until an unverified factual
  accusation is present.
