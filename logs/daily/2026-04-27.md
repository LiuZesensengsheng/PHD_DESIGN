## Candidate Long-Term Memory

- `evaluation_autonomous_design_model_v1` should be treated as a report-only
  orchestration contract over existing cardanalysis surfaces, not a new evaluator.
- The lab's first role-level package skeleton shape is anchor, support, glue, payoff,
  and safety valve.
- `promote` in this lab means human-review queue only; it does not mean default
  recommendation, hard-gate pass, learned promotion, or formal card creation.
- `charge_turns` probes should be band-based and should always name a companion
  `fail_state_floor`; `none_visible` means revise by wording discipline, not by a new
  hard gate.
- The `charge_turns` evidence bundle handoff should use missing-section wording:
  missing fun/health means `not_evaluated`, missing compression means reachability
  unknown, and missing design iteration means no reject/revise/promote wording yet.
- `charge_turns` revision actions should address identity swallowing before payoff
  ambition; a valid revision note names one parameter or one package role to adjust.
- `charge_turns` package-skeleton readiness should check anchor/support/glue/payoff/
  safety-valve roles before using review-ready wording; payoff-only skeletons stay in
  revision.
- `charge_turns` review should keep parameter bands, foundation dependencies,
  fail-state floors, compression assumptions, and evidence conflicts visible instead
  of collapsing them into a hidden score.
- The lab vocabulary has a first transfer example for `discard_outlet_count`; discard
  must prove whether it is an identity, cost, trigger, resource, or release choice
  rather than generic hand cleanup.
- `evaluation_autonomous_design_model_v1` is now an implemented report-only
  orchestration surface; it consumes existing summaries and outputs advisory wording,
  package readiness, evidence gaps, conflicts, and next report requests only.
