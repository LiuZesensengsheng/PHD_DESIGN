## Current Status

- Thesis judgment no longer fabricates a player-facing reject result when the underlying reviewer-slot machinery fails.
- Meeting event selection now respects its own cooldown / once-per-run / condition filters even when the candidate set becomes empty.
- Broken meeting condition payloads now fail closed instead of leaking into runtime as eligible events.
- DDL snake-step failures now surface as actual errors instead of silently letting the turn continue with stale geometry.
- Campaign turn advance now rolls back its core mutations when structural turn resolution aborts mid-flight.
- Trait purchases and retryable event choices no longer pretend success after persistence/effect failures.
- Stress judgment failure no longer leaves combat running in a corrupted, unresolved state.
- The repo now has a single default smoke entrypoint for multi-area changes instead of relying on implicit knowledge of which local gates to combine.
- The project-level toolchain contract now clearly targets Python 3.11 and no longer depends on Poetry or `poetry.lock`.
- The first combat-analysis design/model entrypoint now lives under
  `tools/combat_analysis/` instead of being treated as ordinary design notes under
  `docs/design/`.
- The combat-analysis subsystem now reaches one step further:
  - `analysis snapshot -> balance findings -> tuning suggestions`
  - `deck context + candidates -> draft recommendation + risk warnings`
  - `project cards_generated.json -> projected packs/decks`
- The subsystem can now also read an audited STS reference seed and explicitly record
  where external card semantics remain only partially vectorized.
- The subsystem now has a reference-report pipeline:
  - curated STS sample/negative suites -> stable benchmark -> snapshot JSON ->
    markdown/html renderers
  - the current Ironclad suite now covers `12` positive archetype shells plus `12`
    fragmented negative mixes, plus `9` near-neighbor card-pair checks
  - the generated HTML stays as one self-contained file while still letting the user
    switch between benchmark summary, density scatter, pool distributions, and sample
    detail views, including a dedicated near-neighbor comparison table
