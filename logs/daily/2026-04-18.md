## Later Work: Campaign Lifecycle Machine RFC

- Landed a new architecture RFC for campaign lifecycle sequencing:
  - `docs/development/CAMPAIGN_LIFECYCLE_MACHINE_V1.md`
- The RFC recommends:
  - outer lifecycle as a small state machine
  - explicit ordered steps inside each phase
  - timing windows for future trait/event reactions
  - no generic "everything is a node" campaign rewrite
- The proposed first migration order is:
  - `TURN_TRANSITION`
  - `TURN_INTERRUPTS`
  - `RETURN_RESOLUTION`
- The RFC also freezes a naming rule for this work:
  - use `CampaignLifecyclePhase` / `CampaignLifecycleStep` /
    `CampaignTimingWindow`
  - avoid the generic term `Node` to prevent collision with old campaign-map
    nodes, runtime UI nodes, and narrative quest nodes
