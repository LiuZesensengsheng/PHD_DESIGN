## Current Blockers

- `fast_card_loop` is still a closed-set ranker:
  - it requires both `current_deck_card_ids` and `candidate_card_ids`
  - it does not yet synthesize new card candidates by itself
- The new synthesis bridge is still bounded and honest:
  - it mutates projection-level skeletons, not runtime card JSON/text
  - exhaust-like tradeoffs are surfaced as explicit synthesis notes/risk notes rather than fully calibrated long-run persistence math
- Numeric balancing is still threshold-only:
  - bootstrap / online / dead-card / energy-strain checks exist
  - the new tuner can only do bounded local search over reviewed projection knobs
  - there is still no full balance oracle, long-run simulation, or autonomous numeric sign-off
- `fun_proxy` still has incomplete coverage for:
  - `matchup_elasticity` as a real blind spot
  - `variance_reliance` as a partial neighbor only
  - `degeneracy_risk` as a partial neighbor only
- The new closure sidecar is still an evaluation harness, not a final judge:
  - fixture replay is contextual and reviewed, but not a hard pass/fail contract for generated variant ids
  - `fun_proxy` still cannot honestly close `matchup_elasticity`, and only partially neighbors `variance_reliance` / `degeneracy_risk`
- `Design Loop v1` and the fast-lane workflows still depend on reviewed structured input, not raw card text or autonomous proposal generation.
