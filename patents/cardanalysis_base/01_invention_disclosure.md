# 技术交底书草案：基于压缩语义空间的卡牌游戏内容设计评估方法

版本：v0.2

状态：内部草案，供专利代理人和投资人技术尽调用。未提交专利申请前不宜公开传播。

## 一、拟申请名称

一种基于压缩语义空间的卡牌游戏内容设计评估方法、系统、设备及存储介质。

## 二、技术领域

本发明涉及游戏内容设计辅助、数据处理、推荐排序和计算机辅助平衡验证领域，尤其涉及一种用于卡牌构筑类游戏的卡牌、卡组、敌人和遭遇内容的结构化评估方法。

## 三、背景问题

卡牌构筑类游戏的内容设计通常依赖设计师经验、玩家反馈、少量测试和人工表格。该方式存在以下问题：

1. 单张卡牌强度、卡组结构、敌人压力和关卡遭遇之间存在强耦合，人工评估容易只看局部。
2. 卡牌文本、数值、机制、触发条件和构筑角色之间缺乏统一表示，难以自动比较不同卡牌和不同卡组。
3. 玩家论坛或测试数据中存在正反馈、负反馈、误报、近邻样本和反事实样本，但传统流程难以把它们稳定转化为可训练、可审计的设计依据。
4. 使用通用大模型直接生成卡牌时，容易出现看似合理但结构不闭合、过度离散、误判构筑角色或绕过平衡约束的问题。
5. 敌人设计与卡组设计常常异步进行，导致敌人压力不能精确验证卡组的结构缺口、节奏窗口和决策空间。

因此，需要一种可解释、可审计、可迭代的卡牌游戏内容设计评估方法，将卡牌、卡组、敌人和遭遇压缩到统一语义空间中，再进行结构化评估和排序。

## 四、发明目的

本发明的目的在于提供一种基于压缩语义空间的卡牌游戏内容设计评估方法，使系统能够：

1. 将卡牌对象、卡组对象、敌人对象和遭遇对象转换为统一的机器可处理语义表示。
2. 识别卡组身份、主轴、辅助包、结构缺口、资源窗口和状态覆盖情况。
3. 将敌人或遭遇转换为压力向量、时机窗口、干扰标签和失败模式。
4. 基于卡组结构与敌人压力生成候选卡牌或候选调整的评估结果。
5. 结合正样本、负样本、近邻样本、误报样本和人工审查标签生成可训练的排序数据。
6. 将规则约束、学习排序和人工审核门控分离，使系统既能提高设计效率，又能保留可审计边界。

## 五、核心技术方案

本发明包括以下步骤。

### 1. 内容对象输入

系统接收至少一种内容对象：

- 卡牌对象：包括费用、类型、效果、触发条件、数值、风险说明和来源信息。
- 卡组对象：包括卡牌集合、构筑主题、当前构筑阶段和样本来源。
- 敌人对象：包括敌人行为、压力类型、节奏窗口、干扰方式和答案偏好。
- 遭遇对象：包括多个敌人的组合、阶段变化、压力叠加和关卡目标。

### 2. 压缩语义投影

系统将卡牌投影为包含以下要素的卡牌语义表示：

- 成本向量：能量消耗、手牌消耗、节奏延迟、条件严格度、风险波动等。
- 增益向量：输出、生存、资源、引擎、选择、控制等。
- 状态亲和向量：启动、过渡、在线、转化、恢复等设计状态。
- 角色标签：启动件、桥接件、收益件、转化件、稳定件、工具件、终结件等。
- 关系边：生产、消耗、放大、转化、依赖、稳定等机制关系。
- 风险与不确定注记：未映射效果、过高方差、资源压力、条件过窄等。

卡组投影基于卡牌语义表示聚合形成：

- 主导机制包和辅助机制包。
- 主导能力轴。
- 当前覆盖的设计状态。
- 结构缺口。
- 身份置信度。
- 局部密度、交叉密度、死牌压力、资源压力等指标。

敌人和遭遇投影形成：

- 压力向量。
- 压力标签和干扰标签。
- 时机窗口和节奏说明。
- 失败模式。
- 答案偏好。
- 多敌人或多阶段压力叠加关系。

### 3. 结构评估

系统根据压缩语义表示生成评估结果，包括：

- 卡组身份评估。
- 构筑阶段覆盖评估。
- 卡牌与卡组结构缺口匹配评估。
- 卡组对敌人压力的答案覆盖评估。
- 候选卡牌对缺口、压力和构筑主轴的贡献评估。
- 风险评估，包括资源过载、死牌增加、机制离散、过早收益件、过窄条件等。

### 4. 候选内容排序

系统可以对候选卡牌或候选设计变体进行排序。排序依据包括：

- 与当前卡组主轴的匹配程度。
- 对结构缺口的修复程度。
- 对敌人压力的答案覆盖程度。
- 对设计状态覆盖的改善程度。
- 风险与不确定注记。
- 与已审查正样本、负样本和近邻样本的相对位置。

### 5. 可训练样本构造

系统将人工审查过的案例转换为 pairwise 排序样本。样本类型包括：

- 期望恢复项优于非结构项。
- 期望恢复项优于负近邻。
- 期望锚点优于误报锚点。
- 期望候选优于误报候选。
- 期望 pick 优于上下文中较差 pick。

该样本构造可以用于训练或校验学习排序模型。学习排序模型只作为重排序或影子比较层，不直接绕过规则约束和人工审核。

### 6. 审计和报告输出

系统输出至少一种审计结果：

- 卡组结构报告。
- 敌人压力匹配报告。
- 候选排序报告。
- 样本导出报告。
- 学习排序比较报告。
- 风险和不确定性报告。

报告中保留投影依据、匹配理由、风险注记、负样本对比和人工审核状态。

## 六、与现有技术的区别

本发明不是简单地对卡牌进行数值评分，也不是直接使用大模型生成卡牌文本。区别包括：

1. 使用统一压缩语义空间同时表示卡牌、卡组、敌人和遭遇。
2. 将卡牌设计状态、构筑角色和机制关系作为一等特征，而不是仅使用攻击、防御、费用等表层数值。
3. 通过正样本、负样本、误报样本和近邻样本构造 pairwise 排序监督。
4. 将规则约束、学习排序和人工审查门控分离，提高可解释性和安全性。
5. 同步评估卡组结构与敌人压力，使卡牌设计、卡组设计和敌人设计可以在同一评估框架中迭代。

## 七、已实现证据

当前项目中已有实现证据包括：

- `tools/combat_analysis` 下存在 101 个 Python 文件。
- `tests/fixtures/combat_analysis` 下存在 349 个 fixture 文件，其中 330 个 JSON。
- 已有乐趣 benchmark、检索 benchmark、pick ranking benchmark 和外部参考卡牌库 holdout benchmark。
- 聚焦测试已通过：29 passed。
- 外部参考卡牌库 holdout ranking export 当前覆盖 28 个 reviewed cases，168 个 feature rows，56 个 pairwise examples。
- Reviewed retrieval ranking export 覆盖 27 个 cases，810 个 anchor rows，1253 个 candidate rows，238 个 pairwise examples。
- Pick ranking export 覆盖 24 个 cases，93 个 feature rows，111 个 pairwise examples。

## 八、建议保护点

建议至少保护以下技术点：

1. 将卡牌、卡组、敌人和遭遇投影到统一压缩语义空间的方法。
2. 基于成本向量、增益向量、状态亲和向量、角色标签和关系边生成卡牌语义表示的方法。
3. 基于卡牌语义聚合识别卡组身份、结构缺口和状态覆盖的方法。
4. 基于敌人压力向量、时机窗口和答案偏好进行匹配评估的方法。
5. 基于正负样本和误报样本构造 pairwise 排序监督的方法。
6. 将规则约束、学习排序和人工审核门控分离的设计评估流程。
7. 输出可审计报告和不确定性注记的闭环方法。

## 九、分案和后续专利建议

本底座案建议聚焦“压缩语义空间 + 结构评估 + 候选排序 + 审计输出”。以下内容可以在本案中作为从属技术点出现，但不建议在本案中过度展开，以便保留后续独立申请空间：

1. 乐趣代理指标和乐趣校准方法。
2. 基于正负样本、近邻样本和误报样本的学习排序方法。
3. 规则约束候选生成和变异空间搜索方法。
4. 基于卡组结构缺口反推敌人压力设计的方法。
5. 面向具体项目的 AI 内容生产工作流和人机协同流程。
