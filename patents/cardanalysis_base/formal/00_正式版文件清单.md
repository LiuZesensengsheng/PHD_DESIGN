# 正式版文件清单

本目录存放 CardAnalysis 底座发明专利的正式版准备文件。

范围说明：本底座案聚焦压缩语义空间、结构评估、曲线分析、压力匹配、候选差分评估、学习排序门控和评估流程多关系约束图，不将“乐趣指标”或“超额收益感”作为必要技术特征；相关指标可在后续分案或独立申请中展开。

## 辅助文件

- `05_发明专利请求书信息模板.md`：用于填写请求书前整理申请人、发明人和联系人信息，不作为正式提交文本。
- `06_自审记录与分案建议.md`：用于内部记录本底座案的自审结论、剩余风险和后续分案方向，不作为正式提交文本。
- `build_word_documents.py`：从正式版 Markdown 生成 Word 文件，并将附图绘制为 PNG 后插入 Word。
- `archive/drawings_legacy_svg/`：保留早期 5 图 SVG 图稿，仅用于追溯；当前正式附图以 `generated_images/` 中的 9 张 PNG 为准。

## Word 输出

Word 文件位于 `word/`：

- `01_权利要求书.docx`
- `02_说明书.docx`
- `02_说明书_带附图预览.docx`
- `03_说明书摘要.docx`
- `04_说明书附图.docx`
- `CardAnalysis_底座发明专利_正式版合并稿.docx`
- `CardAnalysis_底座发明专利_图文阅读版.docx`

说明：正式拆分版中，`02_说明书.docx` 不嵌入附图，附图集中在 `04_说明书附图.docx`；阅读版和带附图预览版会把图插入正文附近，便于内部评审或对外沟通。

## 重新生成

```powershell
py -3.11 docs/patents/cardanalysis_base/formal/build_word_documents.py
```

## 摘要附图

建议指定图1作为摘要附图。

## 提交前处理

1. 核对申请人、发明人、联系人和费用减缴信息。
2. 确认权利要求书、说明书、摘要、附图说明和附图编号一致。
3. 按专利业务办理系统要求上传 Word、图片或 PDF 格式文件。
4. 提交前确认没有公开内部样本、模型参数、阈值、prompt 或未公开游戏内容。
