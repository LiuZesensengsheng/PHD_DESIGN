# 说明书草案：基于压缩语义空间的卡牌游戏内容设计评估方法

版本：v0.2

说明：本文件为说明书草案，需由专利代理人按正式格式和检索结果加工。

## 发明名称

一种基于压缩语义空间的卡牌游戏内容设计评估方法、系统、设备及存储介质。

## 技术领域

本发明涉及计算机辅助游戏设计、数字内容评估、推荐排序和数据处理技术领域，尤其涉及一种用于卡牌构筑类游戏内容设计的压缩语义表示、结构评估、候选排序和审计输出方法。

## 术语定义

为便于理解，本说明书中的部分术语定义如下。

压缩语义空间：指将不同类型的游戏内容对象转换为可由计算设备处理的向量、标签集合和关系边集合的统一表示空间。

卡牌对象：指具有费用、类型、效果、触发条件、目标或来源信息的数字内容对象。

卡组对象：指由多个卡牌对象构成的组合式数字内容对象。

敌人对象：指具有压力、干扰、时机窗口、失败模式或答案偏好的数字内容对象。

遭遇对象：指由一个或多个敌人对象以及阶段信息构成的组合式数字内容对象。

结构缺口：指卡组对象在设计状态、构筑角色、资源窗口、能力轴或敌人压力答案方面的不足。

候选贡献表示：指候选内容对象对结构缺口、敌人压力和当前构筑身份的补足或改善程度。

风险表示：指候选内容对象可能引入的资源压力、死牌压力、机制离散、条件过窄、过高方差或投影缺口。

可审计报告：指保留投影依据、计算理由、风险注记和不确定性说明的机器生成报告。

## 背景技术

卡牌构筑类游戏通常包含大量卡牌、构筑路线、敌人行为和遭遇组合。卡牌内容的合理性不仅取决于单卡数值，还取决于该卡牌在卡组中的角色、与其他卡牌的机制关系、所处构筑阶段、面对敌人压力时的答案能力，以及玩家在不同局面下是否拥有清晰的决策空间。

现有设计流程多依赖人工经验、测试反馈和表格管理。该流程存在如下缺陷：

1. 评估粒度不统一。单卡、卡组、敌人和遭遇经常被分开评估，缺少统一的数据结构。
2. 语义损失明显。费用、伤害、抽牌等表层数值不能完整表达启动件、桥接件、收益件、稳定件等构筑角色。
3. 样本利用不足。玩家反馈、论坛讨论和人工评审中包含正样本、负样本、近邻样本和误报样本，但难以转化为稳定的训练和验证数据。
4. 生成结果不可审计。直接使用文本生成模型生成卡牌时，设计理由、结构风险和负样本边界不容易被复核。
5. 敌人设计滞后。敌人压力常常不能针对卡组结构缺口、时机窗口和答案密度进行同步验证。

因此，需要一种能够将卡牌、卡组、敌人和遭遇映射到统一语义空间中的计算机实现方法，使系统可以进行结构化评估、排序、审计和迭代。

## 发明内容

### 要解决的技术问题

本发明要解决的技术问题是：如何在计算机系统中统一表示卡牌游戏内容对象，并基于该统一表示对卡牌、卡组、敌人和遭遇之间的结构关系进行可审计评估。

### 技术方案

为解决上述问题，本发明提供一种基于压缩语义空间的卡牌游戏内容设计评估方法，包括：

1. 获取待评估内容对象。
2. 将内容对象投影为压缩语义表示。
3. 基于压缩语义表示生成结构评估结果。
4. 基于结构评估结果对候选内容对象进行排序或评分。
5. 输出包含投影依据、评估理由、排序结果和风险注记的可审计报告。

在一种实施方式中，内容对象包括卡牌对象、卡组对象、敌人对象和遭遇对象。

在一种实施方式中，卡牌对象的压缩语义表示包括成本向量、收益向量、状态亲和向量、角色标签、关系边和风险注记。

在一种实施方式中，卡组对象的结构评估结果包括主导机制包、辅助机制包、主导能力轴、设计状态覆盖、结构缺口、身份置信度、局部密度、交叉密度、死牌压力和资源压力。

在一种实施方式中，敌人对象和遭遇对象的压缩语义表示包括压力向量、压力标签、干扰标签、时机窗口、失败模式和答案偏好。

在一种实施方式中，系统基于经过人工审查的正样本、负样本、近邻样本或误报样本构造成对排序样本，并使用该成对排序样本训练或验证学习排序模型。所述学习排序模型用于重排序或影子比较，且不直接绕过规则约束和人工审核门控。

### 有益效果

与现有技术相比，本发明至少具有以下有益效果：

1. 将卡牌、卡组、敌人和遭遇统一为机器可处理的压缩语义表示，降低人工评估中的语义不一致。
2. 能够识别卡组结构缺口、状态覆盖不足、资源压力和敌人压力不匹配等问题。
3. 能够将正负样本和误报样本转化为可训练的排序监督，提高样本复用能力。
4. 能够输出可审计报告，保留投影依据、排序理由和不确定性注记。
5. 能够将规则评估、学习排序和人工审核分离，降低黑箱生成内容直接进入设计流程的风险。

在计算机处理层面，本发明通过将异构内容对象映射为固定维度向量、有限字典向量、标签集合和关系边集合，减少候选比较时的自由文本解析量，使候选排序、样本构造和审计输出能够在统一数据结构上执行。

## 具体实施方式

以下结合实施例对本发明进行说明。应当理解，以下实施例用于解释本发明，而非限制本发明的保护范围。

### 实施例一：卡牌对象的压缩语义投影

计算设备接收卡牌对象。卡牌对象可以包括卡牌名称、费用、类型、效果描述、数值字段、触发条件、作用目标和来源标识。

计算设备对卡牌对象进行结构化解析，并生成卡牌压缩语义表示。该表示包括：

- 成本向量，用于表示能量消耗、手牌消耗、节奏延迟、条件严格度和风险波动。
- 收益向量，用于表示输出、生存、资源、引擎、选择和控制。
- 状态亲和向量，用于表示该卡牌在启动、过渡、在线、转化和恢复阶段的作用。
- 角色标签，用于表示该卡牌是启动件、桥接件、收益件、转化件、稳定件、工具件或终结件。
- 关系边，用于表示该卡牌生产、消耗、放大、转化、依赖或稳定其他机制。
- 风险注记，用于表示未映射效果、过高方差、条件过窄、资源压力或投影缺口。

通过上述表示，系统不只保留卡牌的表层数值，还保留其在构筑结构中的功能位置。

在一种实现方式中，字段归一化模块110将卡牌对象中的自由文本字段、枚举字段和数值字段转换为规范化内容数据。压缩语义投影模块120基于规范化内容数据生成成本向量、收益向量、状态亲和向量、角色标签集合、关系边集合和风险表示。上述向量可以是固定维度向量，也可以是有限字典向量。

例如，当卡牌对象具有延迟收益和条件触发信息时，字段归一化模块110将该条件触发信息写入规范化条件字段；压缩语义投影模块120根据该规范化条件字段提高条件严格度维度，并在风险表示中加入条件过窄或高方差信号。

### 实施例二：卡组结构评估

计算设备接收一组卡牌对象，并分别生成每张卡牌的压缩语义表示。随后对多个卡牌语义表示进行聚合，得到卡组对象的结构表示。

聚合过程包括：

1. 统计卡组内机制包的分布，确定主导机制包和辅助机制包。
2. 统计收益向量中的主要能力轴，确定主导能力轴。
3. 汇总状态亲和向量，确定启动、过渡、在线、转化和恢复阶段的覆盖程度。
4. 根据角色标签分布识别缺失或不足的构筑角色。
5. 根据机制包集中度、角色覆盖和状态覆盖生成身份置信度。
6. 根据高费用、条件卡、孤立收益件和资源不足情况生成死牌压力或资源压力。

输出的卡组结构评估结果可用于判断该卡组是否具有清晰构筑身份，是否存在结构缺口，以及候选卡牌应优先修复何种问题。

### 实施例三：敌人压力匹配

计算设备接收敌人对象或遭遇对象，并生成压力表示。压力表示包括压力向量、压力标签、干扰标签、时机窗口、失败模式和答案偏好。

计算设备根据卡组结构评估结果生成答案画像。例如，答案画像可以包括前期输出、群体处理、生存稳定、资源压缩、选择能力、控制能力、长期成长能力等。

计算设备将答案画像与压力表示比较，生成：

- 缺失答案。
- 薄弱答案。
- 时机风险。
- 结构风险。
- 候选转向建议。

该过程使敌人设计和卡组设计处于同一评估空间中。

在一种实现方式中，敌人压力分析模块140根据敌人对象的行为模式生成压力向量，并根据压力向量与卡组答案画像之间的差异生成匹配缺口。匹配缺口可以被候选评估模块150用于判断候选卡牌是否能够补足敌人压力。

### 实施例四：候选内容排序

计算设备获取候选内容对象。候选内容对象可以是候选卡牌、候选卡牌变体、候选卡组调整、候选敌人压力配置或候选遭遇配置。

计算设备对候选内容对象进行压缩语义投影，并基于当前卡组结构评估和敌人压力匹配结果进行排序。

排序特征可以包括：

- 候选对象与主导机制包的匹配程度。
- 候选对象与辅助机制包的匹配程度。
- 候选对象对结构缺口的修复程度。
- 候选对象对薄弱设计状态的补足程度。
- 候选对象对敌人压力或遭遇压力的答案覆盖程度。
- 候选对象引入的死牌压力、资源压力和不确定性。

排序结果可以用于推荐候选卡牌、筛选候选设计变体或提示设计师进行人工复核。

在一种实现方式中，候选评估模块150将候选内容对象的候选贡献表示和风险表示组合为候选排序特征。候选排序特征可以输入基于规则的排序函数，也可以输入学习排序模块170进行重排序或影子比较。

### 实施例五：成对排序样本构造和学习排序

计算设备获取经过人工审查的案例。案例可以包括：

- 正样本，即被确认为结构上合理或符合目标设计意图的内容对象。
- 负样本，即被确认为结构不合理、误导或风险较高的内容对象。
- 近邻样本，即与目标内容对象相似但存在关键结构差异的对象。
- 误报样本，即被基础规则或检索方法错误排到较前位置的对象。

计算设备将上述案例转换为成对排序样本。每个成对排序样本包括赢家对象、输家对象、样本标签类型和差异特征。

计算设备可以基于成对排序样本训练学习排序模型。学习排序模型输出排序修正值。该排序修正值可以用于候选重排序或影子比较。

在一种实施方式中，系统设有推广门控。若学习排序模型导致已有案例回归，或仅在饱和数据上提升而不能改善目标指标，则不将该模型纳入正式排序路径。

上述方式使学习排序模块170不直接改变基础规则结果，而是通过人工审核门控模块190进行控制，从而保留评估流程的可解释性和可回滚性。

### 实施例六：规则约束候选生成和审计

计算设备可以在规则约束的变异空间中生成候选卡牌变体。变异空间包括：

- 费用调整。
- 缺口补充。
- 收益增强。
- 稳定性修正。

每个候选变体生成后重新进行压缩语义投影，并进入结构评估和排序流程。

系统输出可审计报告。报告包括候选对象的投影结果、排序理由、风险注记、与负样本的差异、学习排序比较结果和人工审核状态。

### 实施例七：系统结构

一种实现本发明的系统可以包括：

- 输入模块，用于获取卡牌、卡组、敌人和遭遇对象。
- 投影模块，用于生成压缩语义表示。
- 分析模块，用于生成卡组结构评估和敌人压力匹配结果。
- 排序模块，用于对候选内容对象进行排序。
- 样本构造模块，用于生成成对排序样本。
- 学习排序模块，用于训练或验证重排序模型。
- 审计模块，用于输出报告和不确定性注记。

各模块可以部署在同一计算设备中，也可以部署在多个服务器或本地工作站中。

### 实施例八：数据结构示例

在一种实施方式中，卡牌压缩语义表示可以包括如下字段：

- `object_id`：对象标识。
- `cost_vector`：成本向量。
- `gain_vector`：收益向量。
- `state_affinity`：设计状态亲和向量。
- `role_tags`：角色标签集合。
- `relation_edges`：机制关系边集合。
- `risk_notes`：风险表示。

在一种实施方式中，卡组结构表示可以包括如下字段：

- `package_weights`：机制包权重。
- `axis_weights`：能力轴权重。
- `state_coverage`：状态覆盖。
- `role_coverage`：角色覆盖。
- `structural_gaps`：结构缺口。
- `resource_pressure`：资源压力。
- `dead_card_pressure`：死牌压力。

在一种实施方式中，敌人压力语义表示可以包括如下字段：

- `pressure_vector`：压力向量。
- `disruption_tags`：干扰标签。
- `timing_windows`：时机窗口。
- `failure_modes`：失败模式。
- `answer_preferences`：答案偏好。

上述数据结构仅为示例，实际实现中可以根据游戏内容类型增加、删除或合并字段。

## 工业实用性

本发明可用于数字卡牌游戏、牌库构筑游戏、战斗构筑游戏、策略游戏或其他具有组合内容设计需求的交互式数字内容系统。通过本发明，可以提高内容设计评估效率、减少误报、提升设计审计能力，并支持卡牌、卡组和敌人内容的同步迭代。
