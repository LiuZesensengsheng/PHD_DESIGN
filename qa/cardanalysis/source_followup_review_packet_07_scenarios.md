## 你不用看机器编号

这份包只给你看人能判断的场景。每条只有一个短审核编号，比如 `07-01`。真实追踪编号放在机器队列里，给 Codex 后续生成 decision shard 时使用；你审核时不用读，也不用关心。

这一轮不是让你判断某个机制好不好，而是帮队列变干净：

1. 哪些待审问题已经足够清楚，值得保留为后续审核提示。
2. 哪些问题还太抽象，需要更多具体例子。
3. 哪些问题容易把 advisory 证据说成正式结论，应该设停止线。
4. 哪些条目像重复、合并项或应该打回重写。

## 怎么填

每条只填：

```text
决定:
信心:
备注:
```

决定可以填字母，也可以填中文：

| 选项 | 中文决定 | 后续处理 |
| -- | -- | -- |
| A | 先收下 | 这个卫生点清楚，值得保留为后续审核提示 |
| B | 需要更多证据 | 方向有用，但还缺具体例子或反例 |
| C | 拒绝过度推断 | 容易把待审材料说得太大，应该设停止线 |
| D | 合并重复 | 和别的条目太像，应该合并 |
| E | 无法判断 | 这条对人不够清楚，先打回重写 |

信心可以填：高 / 中 / 低。

## 重要边界

下面的例子都只是为了帮助审核队列，不是正式卡牌设计，不是正式敌人数据，不会进入运行时，也不会改变现有 UI、存档、capability graph、report-only registry、hard gates、默认生成、learned 或 reranker 行为。

这份审核包本身也不会提升证据等级。所有材料仍然保持：

```text
design_note / human_curated / review_needed / advisory_context_only
```

只有你明确审核后，我才会另建 decision shard。现在这份包只是待审队列。

## 最省力的试填方式

你可以只写：

```text
决定: A

备注: 这个卫生点我能理解，值得保留
```

或者：

```text
决定: E
信心: 高
备注: 这条太抽象，我不想审
```

如果一条让你停顿超过 20 秒，就直接填 E。这个包的目标不是考你，而是发现哪些队列条目还没有翻译成人能判断的形式。
