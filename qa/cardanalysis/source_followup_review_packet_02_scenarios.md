## 你不用看机器编号

这份包只给你看人能判断的场景。每条只有一个很短的审核编号，比如 `02-01`。真正的 source case id 放在机器队列里，给 Codex 后续生成 decision shard 时追踪用；你审核时不用读，也不用关心。

你真正要判断的是：

1. 这个场景在卡牌战斗里能不能想明白。

2. 这个风险或边界是否值得继续收集证据。

3. 这条是不是太抽象、太重复，或者已经越界成正式规则判断。

## 怎么填

每条只填：

```text
决定:
信心:
备注:
```

决定可以填字母，也可以填中文：

| 选项 | 中文决定 | 后续处理 |
| -- | -- | -- |
| A | 先收下 | 值得成为后续 reviewed seed 候选 |
| B | 需要更多证据 | 方向有用，但还缺具体例子或反例 |
| C | 拒绝过度推断 | 说得太大、不安全，暂时不能提升 |
| D | 合并重复 | 和别的条目太像，应该合并 |
| E | 无法判断 | 这条对人不够清楚，先打回重写 |

信心可以填：高 / 中 / 低。

## 术语小字典

* 防御反伤: 把防御、格挡或护盾转成反击、反射、反弹伤害。

* 循环安全: 发现循环要失控时，能不能停得住，并且停得公平。

* 恢复窗口: 高压之后给玩家喘息、修复或重新组织的时机。

* 重定向: 把一次攻击、效果或目标从原目标转到另一个目标。

* 公开搜索: 从公开的卡池、目标池或信息池里寻找结果。

* 状态转移: 把负面或正面状态从一个对象移到另一个对象。

* 召唤过期: 临时召唤物、支援物或衍生物到时间后消失。

* 临时生成: 临时生成卡牌、选项或资源，通常只在短时间内可用。

* 变形池: 从一组可能结果里把一张牌或对象变成另一个对象。

## 最省力的试填方式

你可以只写：


或者：

```text
决定: E
```

如果一条让你停顿超过 20 秒，就直接填 E。审核包的目标不是考你，而是发现哪些证据还没有翻译成人能判断的形式。
