## 你不用理解机器编号

像 `human_curated_chain_fatigue_loop_vs_combo_sample_pair_followup_case_v1` 这种编号只是给 Codex 和测试脚本追踪用的。你审核时不用读它，也不用理解它。

你真正要看的只有三件事：

1. 这个“假想游戏情境”你能不能理解。

2. 这个问题是否值得以后继续收集证据。

3. 这条是不是太抽象、太重复、或者你根本无法判断。

## 怎么填

每条只填：

```text
决定:
信心:
备注:
```

决定可以填中文，也可以填字母：

| 选项 | 中文决定   | 我后续怎么处理                   |
| -- | ------ | ------------------------- |
| A  | 先收下    | 这条值得成为后续 reviewed seed 候选 |
| B  | 需要更多证据 | 方向有用，但还缺具体例子或反例           |
| C  | 拒绝过度推断 | 说得太大、不安全，暂时不能提升           |
| D  | 合并重复   | 和别的条目太像，应该合并              |
| E  | 无法判断   | 这条对人不够清楚，先打回重写            |

信心可以填：高 / 中 / 低。

你完全可以这样填：

```text
决定: E
信心: 高
备注: 我看不懂这个问题在游戏里对应什么
```

这不是失败，反而是很有价值的审核结果。它说明我们的证据包还没有做到“人能审”。

## 术语小字典

* 连锁疲劳: 连续触发、反复循环、越打越累的问题。

* 抽牌热度: 抽牌、换牌、重抽、手牌变多带来的注意力压力。

* 能量不稳定: 这回合爆发，下回合欠债或卡死。

* 污染市场: 现在拿收益，以后承担废牌、状态牌、延迟代价。

* 套件崩塌: 一组牌缺启动、缺收益、缺兜底，导致整套机制失灵。

## 最省力的试填方式

你可以先只填 3 条：

```text
决定: A
信心: 中
备注: 这个问题我能理解，值得保留
```

或者：

```text
决定: E
信心: 高
备注: 这个太抽象，我不知道游戏里对应什么
```

如果一条让你停顿超过 20 秒，就直接填 E。审核包的目标不是考你，而是暴露哪些证据还没有被翻译成人能判断的形式。
