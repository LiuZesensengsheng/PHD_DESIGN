## 你不用看机器编号

这份包只给你看人能判断的场景。每条只有一个短审核编号，比如 `05-01`。真实追踪编号放在机器队列里，给 Codex 后续生成 decision shard 时使用；你审核时不用读，也不用关心。

你真正要判断的是：

1. 这个外部参考场景能不能想明白。
2. 这个问题是否值得以后继续收集项目内证据或反例。
3. 这条是否只是外部启发，不能被写成 reviewed 结论、正式规则、默认生成依据或运行时行为。

## 怎么填

每条只填：

```text
决定:
信心:
备注:
```

决定可以填字母，也可以填中文：

| 选项 | 中文决定 | 后续处理 |
| -- | -- | -- |
| A | 先收下 | 值得成为后续 reviewed seed 候选 |
| B | 需要更多证据 | 方向有用，但还缺具体例子或反例 |
| C | 拒绝过度推断 | 说得太大、不安全，暂时不能提升 |
| D | 合并重复 | 和别的条目太像，应该合并 |
| E | 无法判断 | 这条对人不够清晰，先打回重写 |

信心可以填：高 / 中 / 低。

下面的例子都只是为了帮助审核，不是正式卡牌设计，不是正式敌人数据，不会进入运行时，也不会改变现有 UI、存档、capability graph、report-only registry、hard gates、默认生成、learned 或 reranker 行为。

## 术语小字典

* 外部参考: 从别的游戏、公开讨论、经验模型或 Codex 整理中来的启发，只能帮我们提出问题。
* readable: 玩家能在行动前看懂压力、代价、倒计时或边界。
* soft cap: 不是硬禁止，而是用逐渐增加的压力提醒玩家不要无限重复。
* partial value: 即使没有完美触发，也保留一点价值，避免空转或全损。
* negative control: 明确记录“这里不应该成立”的反例，防止概念变得过宽。
* authority quarantine: 把外部参考隔离在 advisory 层，避免它变成 reviewed 结论。
