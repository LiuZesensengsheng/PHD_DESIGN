## 你不用看机器编号

这份包只给你看人能判断的重写问题。每条只有一个短审核编号，比如 `08-01`。它背后会回指到旧审核包里的某一条，但你不用读旧编号，也不用看机器 case id。

这一轮不是判断机制是否好，而是判断“这条待审材料是否需要改写”：

1. 是否缺少具体例子。
2. 是否缺少反例或停止线。
3. 是否容易把 advisory 材料说成正式结论。
4. 是否可以保留当前写法，还是应该打回重写。

## 怎么填

每条只填：

```text
决定:
信心:
备注:
```

决定可以填字母，也可以填中文：

| 选项 | 中文决定 | 后续处理 |
| -- | -- | -- |
| A | 当前写法可保留 | 不需要重写，可回到原待审队列 |
| B | 需要更多例子 | 保留方向，但补具体样本或反例 |
| C | 需要停止线 | 保留方向，但必须写清楚不能推断到哪里 |
| D | 合并重复 | 和别的条目太像，应合并 |
| E | 打回重写 | 当前写法不适合人工审核 |

信心可以填：高 / 中 / 低。

## 重要边界

这份包仍然只是待审队列，不是 decision shard。所有材料仍然保持：

```text
review_needed / advisory_context_only
```

不会生成正式卡牌，不会改运行时、正式数据、UI、存档、capability graph、report-only registry、hard gates、默认生成、learned 或 reranker 行为。
