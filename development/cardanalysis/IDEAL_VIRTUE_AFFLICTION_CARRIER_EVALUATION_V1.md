# Ideal Virtue Affliction Carrier Evaluation V1

## Problem

项目已经确定 `理想`、`美德`、`折磨` 是核心表达，但仍需要决定它们由什么玩法结构承载。

当前危险点不是主题不成立，而是承载方式可能把内容量、玩家理解成本、调参成本推到不可控：

- 如果每个理想都要求专属卡牌池，卡牌数量会迅速膨胀到难以设计、测试和让玩家理解。
- 如果理想直接限制玩家出牌，容易变成道德审判或隐藏扣分，而不是好玩的构筑选择。
- 如果理想只变成简单数值特质，又可能失去卡牌系统联动，成为叙事上好听但玩法上很淡的标签。

本文件是讨论模型，不是最终拍板。它用于把几种方案和参考游戏放进同一张评分表，帮助继续收敛。

## Constraints

- `evaluation_mode = report_only`
- `authority_boundary = advisory_context_only`
- `hard_gate_impact = none`
- 不在本文件内设计正式卡牌、不写运行时数据、不改变现有战斗或战役逻辑。
- 不把 `理想` 做成每回合隐藏审计，也不默认让理想限制玩家选卡。
- 核心战斗仍应优先保护类似 `draw cards -> spend energy -> play cards -> read enemy intent -> choose rewards` 的低摩擦循环。
- 允许 `理想`、`美德`、`折磨` 在构筑、事件、特质、压力阈值、奖励交换、战役选择中产生影响，但应避免把复杂度塞进每张卡和每个回合。

## Complexity

### Essential Complexity

| 必要复杂度 | 说明 |
| --- | --- |
| 价值身份 | 玩家需要感觉自己的研究者、学科、人格、经历和理想有关系。 |
| 机械后果 | 理想不能只是文本标签，必须至少在数值、奖励、事件、压力、特质或卡牌构筑中留下痕迹。 |
| 内部张力 | 玩家应该在少数清晰变量里面对收益之间的拉扯，并看见可能的连锁收益或连锁崩溃。 |
| 长期记忆 | 美德/折磨/理想最好能在一局或一次战役里留下持续影响，而不是只在单次事件里闪一下。 |
| 局部可玩 | 即使完整版还没做完，最小切片也应该已经能玩出选择感。 |

### Accidental Complexity

| 偶发复杂度 | 为什么危险 |
| --- | --- |
| `75 * 5` 或更高的卡池乘法 | 内容量、平衡量、测试量、UI 解释量同时爆炸。 |
| 每张卡都绑定理想版本 | 容易让玩家先读阵营再读效果，降低卡牌直觉。 |
| 隐藏行为审计 | 玩家不知道自己为什么被奖励或惩罚，会把系统理解成道德裁判。 |
| 过多一次性无关选择 | 如果理想、人格、经历只是分散选择堆积，最后会像属性表，不像玩法系统。 |
| 全量仿照参考游戏 | 杀戮尖塔、暗黑地牢、文明、冰汽时代解决的是不同问题，只能借结构，不能整套搬。 |

## Scoring Model

### 分数方向

每项原始分都是 `1-5`：

| 维度 | 5 分含义 | 1 分含义 | 计分方向 |
| --- | --- | --- | --- |
| 完整版潜力 | 做完整后上限很高，能成为核心卖点 | 做完整也只是边缘系统 | 越高越好 |
| 设计复杂度难度 | 内容、平衡、实现、维护都很重 | 很容易做和维护 | 原始分越高越危险 |
| 玩家心智负担 | 玩家需要理解很多规则、例外或长期状态 | 玩家几乎一眼能懂 | 原始分越高越危险 |
| 叙事自恰 | 理想/人格/经历/美德/折磨关系自然 | 主题和机制像硬贴 | 越高越好 |
| 内部张力 | 少数变量能产生连锁与收益取舍，且每条路都有正向诱因 | 只是惩罚、限制、加减数值或变量堆叠 | 越高越好 |
| 部分好玩 | 小切片也能形成可玩循环 | 只有完整版完成才可能好玩 | 越高越好 |
| 先例可循 | 有强参考模型可拆解 | 几乎没有可借鉴结构 | 越高越好 |

### 总分公式

`设计复杂度难度` 和 `玩家心智负担` 是风险轴，因此总分使用反向后的可控性：

```text
可控复杂度 = 6 - 设计复杂度难度
可控心智负担 = 6 - 玩家心智负担

总分 =
  完整版潜力 / 5 * 15
  + 可控复杂度 / 5 * 20
  + 可控心智负担 / 5 * 15
  + 叙事自恰 / 5 * 15
  + 内部张力 / 5 * 15
  + 部分好玩 / 5 * 15
  + 先例可循 / 5 * 5
```

权重故意偏向 `可控复杂度`、`可控心智负担` 和 `部分好玩`，因为当前最大风险是做不完、玩不明白、局部版本不好玩。

### 内部张力的评价依据

这里的 `内部张力` 不是指系统复杂，也不是指惩罚重，更不是指限制玩家。它应该更克制：用少数变量制造收益之间的拉扯，让玩家知道自己在买什么、赌什么、放弃什么。

| 表现形式 | 好的表现 | 坏的表现 | 设计要求 |
| --- | --- | --- | --- |
| 连锁崩溃 / 连锁收益 | 一个选择会打开后续机会或风险，例如压力接近阈值后可能触发美德反转，也可能进入折磨连锁。 | 随机惩罚突然砸下来，或者连锁收益无脑滚雪球。 | 连锁必须可见、可预判、可干预；玩家要能提前感到“再往前一步会变形”。 |
| 收益的取舍 | 每个选择都先给收益：更多爆发、更稳防御、更好奖励、更快研究、更低压力；代价是购买这个收益的价格。 | 把“张力”理解成扣血、扣资源、扣道德分，导致玩家只是在躲惩罚。 | 核心是收益，不是惩罚；妥协也应该有现实收益，坚持也应该有策略收益。 |
| 决策变量克制 | 一次只让玩家关注 1-2 个关键变量，例如当前收益、未来压力、构筑方向中的两项。 | 同时要求玩家计算理想、人格、经历、美德、折磨、事件、职业、隐藏标签。 | 变量越少，张力越强；变量过多会把张力变成噪音。 |

一句话：好的内部张力让玩家说“两个选择都有赚头，所以难选”；坏的内部张力让玩家说“系统在罚我、管我、考我”。

## Options

| 编号 | 方案 | 简述 |
| --- | --- | --- |
| A | `75理想 * 5卡牌共享` | 每个理想都拥有或改写一大组卡牌内容，形成理想维度的大联动。 |
| B | `职业50 + 5*5理想` | 职业提供主卡池，理想提供较小的专属卡、事件或模块池。 |
| C | `75职业 + 理想特质` | 职业卡池为主体，理想通过人格特质/经历承载，主要影响数值、代价或奖励倾向。 |
| D | `职业卡 + 理想特质 + 美德/折磨事件阈值` | 理想先作为特质影响构筑和数值，美德/折磨通过压力、事件或阈值选择制造戏剧性。 |
| E | `理想主要叙事化` | 理想主要进入剧情、事件文案和结局分支，对卡牌影响很弱。 |
| F | `理想 = 遗物/天赋式玩法插件` | 理想像开局遗物、天赋、法令或核心被动，少量规则改变带来构筑方向。 |
| G | `卡牌行为标签审计` | 每张牌或行动被理想标签评价，系统持续累计一致/背离。 |
| H | `理想 = 通用附魔模板` | 卡牌可以获得带颜色的通用附魔；每个理想 2-4 个模板，附魔数量形成理想倾向，美德强化附魔，折磨污染附魔。 |
| R1 | `杀戮尖塔基线` | 小职业卡池 + 遗物 + 能量回合 + 奖励选择；核心循环极清晰。 |
| R2 | `暗黑地牢基线` | 压力积累 + 美德/折磨阈值 + 长期角色后果；心理状态戏剧性强。 |
| R3 | `文明基线` | 领袖/文明特性 + 科技/政策/路线选择；长期身份强，但局部回合重。 |
| R4 | `冰汽时代基线` | 法典/信仰或秩序路线 + 希望/不满 + 生存危机；价值选择和机械后果紧密绑定。 |

## Reason Table

| 方案/基线 | 分数理由 | 内部张力依据 | 主要风险 | 可借鉴结论 |
| --- | --- | --- | --- | --- |
| A. `75理想 * 5卡牌共享` | 完整版潜力很高，因为理想能深入卡牌文本、组合和构筑；但设计复杂度与玩家心智负担都是最高档，局部版本几乎无法证明好玩。 | 理想专属卡可以做出连锁收益、连锁风险和收益取舍。问题是张力要靠海量内容供给，变量也容易失控。 | 内容生产量失控；平衡验证失控；玩家看到卡池就迷路；一年内很难完成可验证版本。 | 当前不要走“大卡组乘法”。最多保留少量标志性理想牌作为奖励或事件产物。 |
| B. `职业50 + 5*5理想` | 用职业卡托底，理想只做小池，复杂度比 A 低；但仍需要同时维护职业身份和理想身份。 | 职业收益和理想收益可以互相拉扯，例如效率收益、研究收益、压力收益之间取舍。 | 小理想池如果太强会绑架构筑，太弱又像装饰；不同职业和理想组合仍会产生大量边界情况。 | 可以作为中期扩展，不适合作为现在的第一版核心。 |
| C. `75职业 + 理想特质` | 生产最稳：卡牌按职业设计，理想只通过特质改变数值、代价、奖励权重或事件倾向。 | 张力偏弱，因为多数选择会变成“这个收益倍率更适合我吗”。如果没有连锁或阈值，理想很难形成戏剧性。 | 玩法与叙事脱节；理想像开局属性；玩家选完后很快忘记。 | 适合作为底座，但不应是全部。需要搭配事件、阈值或特殊奖励来补张力。 |
| D. `职业卡 + 理想特质 + 美德/折磨事件阈值` | 把高频战斗复杂度留给职业卡，把理想放到低频但高影响的特质、压力、事件和阈值里；总量可控且叙事很顺。 | 玩家可以为了当前收益、未来连锁收益、压力阈值和构筑方向做取舍。张力来自收益的价格，不来自禁止出牌。 | 需要把阈值触发做得清楚；如果随机性太强会显得不公平；如果事件太少会退化成 C。 | 当前最像主线：先做少量理想特质，再接 1-2 个美德/折磨阈值事件。 |
| E. `理想主要叙事化` | 叙事自恰很高，制作也相对轻；但对卡牌核心循环贡献低。 | 张力多在剧情选择里，卡牌战斗里不明显。 | 玩家会觉得理想只是剧情皮肤；重玩动力不足；构筑不能表达角色。 | 可以作为补强层，不应单独承载核心玩法。 |
| F. `理想 = 遗物/天赋式玩法插件` | 类似遗物、天赋或法令，少量规则就能改变出牌和构筑方向；可做小切片，也容易解释。 | 每个理想给一个诱人的收益入口和一个清楚价格，例如更强爆发但更接近压力阈值、更稳防御但奖励成长更慢。 | 设计不好会变成纯数值加成；太强会锁死最优理想；太弱会没有存在感。 | 很适合第一版：每个理想 1 个核心收益、1 个价格、1 个可触发美德/折磨入口。 |
| G. `卡牌行为标签审计` | 理论上能让每张行动都和理想相关，但玩家理解成本极高，且需要大量 UI 与反馈。 | 张力可能存在，但经常变成“我是不是被系统判错了”。 | 隐藏审计、道德裁判感、出牌焦虑、反馈噪音、测试困难。 | 当前应明确列为反例。除非未来做成完全透明且低频的事件复盘，否则不进入主路线。 |
| H. `理想 = 通用附魔模板` | 理想直接落在卡牌上，玩家通过给卡牌附魔形成牌组颜色。每个理想只需要 2-4 个通用模板，内容量远低于每卡五色版本。 | 张力来自附魔选择本身：一张牌只能承载少数附魔，牌组颜色会自然形成主导理想；美德强化该色附魔，折磨污染该色附魔。 | 如果模板太多，会重新变成内容爆炸；如果每张卡都手写附魔，会回到 `75*5`；如果美德/折磨只改数值，会失去戏剧感。 | 当前最推荐。必须坚持“模板化附魔”，第一版每个理想 2-4 个模板，每张卡最多 1 个附魔槽。 |
| R1. 杀戮尖塔基线 | 它证明核心循环必须极清楚：抽牌、能量、敌意图、奖励选择。复杂度主要来自卡牌组合和遗物，而不是规则解释。 | 张力来自当回合生存、长期构筑、稀有奖励、遗物联动和路线风险。 | 叙事身份弱；如果照搬会失去本项目的理想/美德/折磨特色。 | 本项目应学习它的“卡牌主体要先好玩”，以及用遗物式小插件承载构筑方向。 |
| R2. 暗黑地牢基线 | 它证明压力阈值、美德/折磨、长期角色后果能形成强戏剧性。 | 张力来自继续深入的收益、撤退的保全收益、承受压力后的反转收益，以及崩溃连锁风险。 | 高随机和高惩罚如果处理不好，会让玩家觉得失控。 | 本项目适合借“压力阈值结构”，但要降低失控感，让玩家能预防、缓解、利用。 |
| R3. 文明基线 | 它证明长期身份、路线、政策和科技可以形成宏观策略；但它不是高频卡牌战斗模型。 | 张力来自扩张、科技、文化、军事、外交、政策切换和时代目标之间的取舍。 | 系统层太多，局部回合很重；若照搬会进一步推高心智负担。 | 适合借“文明/领袖特性”的身份框架，不适合借完整宏观系统。 |
| R4. 冰汽时代基线 | 它证明价值选择可以直接变成机械后果：法典、希望、不满、生存压力和不可逆承诺互相咬合。 | 张力来自每条法案先提供生存收益，再带来路线承诺、社会反应和后续连锁。 | 如果每次选择都太沉重，会让卡牌游戏节奏变慢；如果全搬法典，会变成城市治理而不是卡牌构筑。 | 适合借“低频高压法案/危机选择”。理想可以像法典承诺一样改变战役规则，但不应审计每次出牌。 |

## Reference Lessons

| 参考 | 对本项目最有用的结构 | 不应照搬的部分 |
| --- | --- | --- |
| 杀戮尖塔 | 小而清晰的战斗核心；卡牌和遗物承担主要乐趣；奖励选择自然塑造构筑。 | 叙事弱表达；不要只做抽象机制而丢掉理想/人格/经历。 |
| 暗黑地牢 | 压力积累、阈值戏剧、美德/折磨分叉、长期后果。 | 高失控随机、高惩罚、玩家命令被频繁拒绝。 |

| 冰汽时代 | 价值选择必须有机械后果；短期生存和长期道德承诺天然冲突；危机让理念变得有重量。 | 把每个选择都做得过重，或把卡牌游戏改造成法典/城市治理模拟。 |

## Risks

| 风险 | 触发信号 | 缓解方式 |
| --- | --- | --- |
| 内容乘法复发 | 开始要求每个理想都有一整套专属卡池。 | 限制第一版为每个理想 1 个核心特质、1 个代价、少量事件或奖励。 |
| 理想变成纯数值 | 玩家选完理想后，战斗中几乎不再感到它存在。 | 给每个理想至少一个可见触发点，例如阈值、事件、奖励交换或特殊遗物。 |
| 道德审判感 | 玩家觉得系统在评价自己“对不对”，而不是给自己策略选择。 | 不做隐藏审计；反馈必须透明；妥协也应有现实价值。 |
| 惩罚伪装成张力 | 玩家主要在避免扣血、扣资源、扣道德分，而不是主动购买收益。 | 每个代价前面必须先有明确收益；文本和 UI 先展示“你获得什么”，再展示“你承担什么价格”。 |
| 决策变量过多 | 玩家每次选择都要同时计算 3 个以上新变量。 | 第一版每个理想只保留 1 个核心收益变量和 1 个价格变量；连锁只围绕这两个变量发生。 |
| 附魔模板膨胀 | 每个理想开始设计 10-20 个附魔，或每张卡都写专属五色版本。 | 每个理想第一版只做 2-4 个通用模板；每张卡最多 1 个附魔槽；附魔适用范围由卡牌类型过滤。 |
| 玩家心智负担过高 | 玩家需要同时读职业、理想、人格、经历、美德、折磨、多种隐藏状态。 | 把高频规则压缩到 1-2 个可见状态；把复杂叙事放在低频事件里。 |
| 局部版本不好玩 | 只有写完大量理想和事件后才可能验证。 | 优先做能独立运转的小切片：职业卡 + 2 个理想色 + 每色 2 个附魔模板 + 1 个压力阈值事件。 |

## Recommendation

优先推进 `H + 压力美德/折磨` 的混合路线：

| 层 | 第一版建议 | 原因 |
| --- | --- | --- |
| 职业/学科 | 继续作为主卡池和主要战斗玩法来源。 | 保护卡牌循环，让局部版本先好玩。 |
| 理想 | 做成带颜色的通用附魔模板。 | 理想直接进入卡牌构筑；每个理想 2-4 个模板即可，不需要每张卡手写多版本。 |
| 附魔槽 | 每张卡最多 1 个附魔槽，少数特殊卡例外。 | 防止读牌和构筑复杂度失控。 |
| 理想强度 | 由牌组中的附魔颜色自然形成。 | 玩家不需要管理五条理想进度；牌组本身就是理想倾向。 |
| 美德/折磨 | 压力爆表后随机触发；美德强化主导色附魔，折磨污染主导色附魔。 | 保留暗黑地牢的戏剧随机，同时让结果和卡牌系统强联动。 |
| 创新点 | 作为论文发表后的中高强度构筑奖励。 | 创新点负责强机制拐点，理想附魔负责牌组风格，美德/折磨负责危机戏剧。 |

第一版可以这样定义目标：

```text
学科卡池负责“每回合好玩”；
通用附魔负责“理想如何进入卡牌”；
附魔颜色负责“牌组正在变成哪种人”；
压力负责“什么时候进入戏剧节点”；
美德负责“让主导色附魔短暂升华”；
折磨负责“让主导色附魔被污染或变得不可靠”；
创新点负责“论文发表后的强构筑拐点”。
```

### 附魔模板边界

第一版推荐范围：

| 项 | 建议 |
| --- | --- |
| 理想数量 | 5 色可以保留。 |
| 每色模板数 | 2-4 个，不超过 4 个。 |
| 总模板数 | 10-20 个，优先从 10-15 个开始。 |
| 每张卡附魔槽 | 默认最多 1 个。 |
| 适用方式 | 附魔模板按卡牌类型过滤，例如攻击、技能、能力、研究、应用。 |
| 理想强度 | `该色附魔数量` 或 `普通=1 / 稀有=2 / 核心=3`。 |
| 美德作用 | 主导色附魔数值提高、额外触发、临时复制、压力回落。 |
| 折磨作用 | 主导色附魔带副作用、概率失效、塞入负面牌、压力额外增长。 |

示例：

| 理想色 | 附魔模板方向 | 美德强化 | 折磨污染 |
| --- | --- | --- | --- |
| 真理 | 抽牌、预见、检索 | 真理附魔额外抽牌或预见敌意图 | 触发真理附魔时加入“怀疑”或增加压力 |
| 荣耀 | 爆发、击杀、连击 | 荣耀附魔击杀后额外减压或再触发 | 荣耀附魔造成更高伤害但额外涨压 |
| 守护 | 格挡、保护、减伤 | 守护附魔额外获得格挡或保护一次压力伤害 | 守护附魔变得保守，例如降低下一张攻击收益 |
| 自由 | 弃牌、重抽、变通 | 自由附魔额外重抽或降低费用 | 自由附魔可能随机弃牌或打乱手牌 |
| 生存 | 恢复、减压、续航 | 生存附魔额外恢复压力或清除负面牌 | 生存附魔变成逃避，例如降低奖励选择或攻击节奏 |

## Counter-Review

这个推荐也有问题：

| 反对点 | 为什么成立 | 需要验证什么 |
| --- | --- | --- |
| `H` 可能变成内容乘法 | 如果每张卡都写 5 种附魔，它会回到不可控的 `75*5`。 | 附魔是否真的来自 10-20 个通用模板，而不是每卡专属文本。 |
| 附魔可能锁死构筑 | 某个颜色模板太强，会让玩家永远选同一色。 | 每个颜色是否至少有 2 种合理打法；附魔是否按卡牌类型限制。 |
| 美德/折磨阈值可能失控 | 阈值如果太随机，会像惩罚；如果太可控，又没戏剧性。 | 玩家是否能预判、推迟、利用或恢复阈值结果。 |
| 折磨污染可能太烦 | 如果折磨频繁破坏玩家操作，会变成负体验。 | 折磨是否仍保留可玩收益或清晰恢复路径。 |

## Decision Summary

| 结论 | 内容 |
| --- | --- |
| 暂不推荐 | A. `75理想 * 5卡牌共享`；G. `卡牌行为标签审计`。 |
| 谨慎保留 | B. `职业50 + 5*5理想` 可以作为未来扩展，但不适合现在主路线。 |
| 可作为底座 | C. `75职业 + 理想特质`，但必须补充事件/阈值，否则太淡。 |
| 旧推荐降级 | F. `理想 = 遗物/天赋式玩法插件` 可以保留为备选，但会和创新点/遗物空间竞争。 |
| 当前推荐 | H. `理想 = 通用附魔模板` + `压力触发美德/折磨`。 |
| 参考优先级 | 卡牌核心学杀戮尖塔；美德/折磨阈值学暗黑地牢；长期身份学文明；价值危机学冰汽时代。 |
| 下一步最小切片 | 选 1 个学科、2 个理想色、每色 2 个附魔模板、20-30 张基础卡、3 个美德、3 个折磨，验证“附魔颜色 + 压力爆表”是否已经好玩。 |
