# 暗黑地牢美德与折磨统计 V1

## 目的

本文件统计《暗黑地牢 1》的美德与折磨系统，重点回答两个问题：

* 美德到底奖励什么？

* 折磨到底损坏什么？

结论先行：

```text
暗黑地牢 1 没有给每个战斗机制配一个对应惩罚。
它用统一的压力阈值制造低频大事件。
美德是纯正向的英雄时刻。
折磨主要损坏角色可靠性、队伍压力、行动控制和远征后勤。
```

## 资料边界

* 本文件是设计参考，不是项目数值模板。

* 本文件只用于学习《暗黑地牢 1》的结构。

* 本文件统计主体结构，并把 DLC/特殊状态单独标记。

### 美德设计解读

《暗黑地牢 1》的美德非常克制：

* 基础美德只有 5 种。

* 每种美德有一个清晰主题：承伤、勇气、专注、力量、活力。

* 美德是纯正向，不混入惩罚。

* 美德的自发行为是帮助玩家，而不是夺走控制。

* 美德的队伍价值很强，尤其是压力缓解和队友强化。

这说明美德不是“某个机制变强”，而是：

```text
角色在压力下变得可靠、鼓舞、能扛、能打、能救场。
```

## 折磨历史权重

| 历史规则                  | 含义                    |
| --------------------- | --------------------- |
| 每种基础折磨初始权重相同。         | 第一次折磨大体平等随机。          |
| 英雄每次获得某种折磨，该折磨未来权重上升。 | 角色会形成“习惯性崩溃方式”。       |
| 美德/折磨的先验检查独立于具体折磨历史。  | 先判定是否美德，再用历史决定具体折磨类型。 |
| 美德历史权重也会增长，但增长幅度低于折磨。 | 角色历史影响未来，但折磨人格更容易积累。  |

这点很重要：

```text
暗黑地牢 1 的“人格”不是开局选一个标签，
而是由角色经历沉淀出来。
玩家记住的不是公式，
而是“这个人上次就这样崩过”。
```

## 设计解读

### 美德奖励什么

| 奖励层  | 暗黑地牢 1 的证据                 | 设计解读             |
| ---- | -------------------------- | ---------------- |
| 可靠性  | 美德不会拒绝命令，反而给正向自发行为。        | 美德首先是可靠性上升。      |
| 队伍稳定 | 勇敢、专注、强力都直接帮队友；进入美德时也缓解压力。 | 美德不是个人强化，而是队伍救场。 |
| 危机转换 | 100 压力本应危险，美德把它变成翻盘窗口。     | 美德的价值来自发生在危机点。   |
| 简单原型 | 5 个美德主题非常直白。               | 美德不需要复杂树或机制绑定。   |

### 折磨损坏什么

| 损坏层  | 暗黑地牢 1 的证据              | 设计解读              |
| ---- | ----------------------- | ----------------- |
| 可靠性  | 强制行为、跳过、随机技能、强制移动。      | 折磨核心是角色不能稳定执行计划。  |
| 队伍信任 | 全队压力、攻击队友、队友行为触发压力发言。   | 折磨会把一个人的崩溃变成队伍问题。 |
| 恢复窗口 | 拒绝治疗、增益、露营、物品、进食、撤退。    | 折磨不只影响战斗，还影响玩家止损。 |
| 生存边际 | 共通最大生命/抗性下降，闪避/命中/伤害变化。 | 折磨让危险事件更容易滚起来。    |
| 探索经济 | 强迫互动奇物、陷阱/奇物压力发言。       | 折磨损坏远征资源管理。       |

### 暗黑地牢 1 没有做什么

| 没有做            | 为什么重要                    |
| -------------- | ------------------------ |
| 没给每个战斗机制单独配代价。 | 避免设计过重，也避免玩家觉得每个技能都被审计。  |
| 没把美德做成构筑路线奖励。  | 美德是危机中的戏剧反转，不是长期天赋树。     |
| 没让折磨只等于数值减益。   | 折磨通过行为、拒绝、队伍压力和探索破坏产生体验。 |
| 没隐藏压力条。        | 压力可见，阈值可预期，失控才可接受。       |

## 对我们项目的启发

| 经验      | 应用到我们                                 |
| ------- | ------------------------------------- |
| 压力是统一入口 | 不要让理想给每个机制配惩罚；压力可以来自战斗、事件、路线和选择。      |
| 美德应纯正向  | 美德应该是低频强正向，不要混入代价。                    |
| 折磨损坏可靠性 | 折磨不必惩罚某个机制；它可以损坏出牌可靠性、奖励选择、事件选项、恢复窗口。 |
| 阈值必须可见  | 玩家需要看到压力条和危机接近。                       |
| 个性来自历史  | 理想/经历可以改变美德/折磨候选权重，而不是每回合评价玩家行为。      |
| 数量要克制   | 《暗黑地牢 1》只有 5 个美德、7 个基础折磨。我们第一版应该更少。   |

## 建议的最小借鉴版本

如果借鉴《暗黑地牢 1》，第一版不应做“大理想机制树”，而应做：

```text
1 个压力条
3 个美德
3 个折磨
少量历史权重
清晰的危机表现
```

示例结构：

| 组件   | 最小版本                            |
| ---- | ------------------------------- |
| 压力   | 战斗受伤、失败事件、强行推进路线会增加。            |
| 决心检定 | 压力满时检查美德/折磨。                    |
| 美德   | 纯正向：减压、抽牌、强化下一张牌、保护一次失败。        |
| 折磨   | 损坏可靠性：随机弃牌、拒绝奖励选项、临时塞入负面牌、压力传染。 |
| 理想影响 | 理想只调整候选权重和表现文本，不给每张牌加审计。        |
| 历史   | 某英雄上次出现的折磨，下一次更容易重复。            |

## 关键结论

《暗黑地牢 1》的统计结论很清楚：

```text
美德 = 危机中的可靠性上升 + 队伍稳定 + 纯正向反转。
折磨 = 可靠性下降 + 压力传染 + 恢复/撤退/资源管理被破坏。
```

它不是：

```text
机制越强，惩罚越强。
```

它是：

```text
压力满后，人会以不同方式变得更可靠或更不可靠。
```
