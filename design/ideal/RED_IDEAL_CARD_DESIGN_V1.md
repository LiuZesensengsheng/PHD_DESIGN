## 红色 · 影响轴 v1（核心：拿来主义连打，短期透支爆发）

### 目标与范围
- 明确红色在本游戏中的核心表达：**积极搜索、拿来就用、回合内爆发、事后短期透支**。
- 围绕【本回合出牌量】与【本回合新抽的牌】定义红色的系统引擎，为后续红色卡牌、敌人和事件设计提供统一口径。
- 本版本暂不与白色指针/排序等机制绑定，只从红色自身性格出发，保证风格独立、边界清晰。

---

### 1. 红色引擎（系统定义）

#### 1.0 总览：主引擎 + 两条支线 + 随机变体
- **主引擎：最新牌连打循环（Fresh + Combo + PlayCount）**  
  - 使用 `cards_played_this_turn`、“本回合新抽的牌（Fresh）”与 `combo_streak` 三个量，定义“这一回合出牌节奏”。  
  - 核心体验：**抽上来就用 → 连打 → 连击伤害变大**，越讲越上头。  
  - 详细规则见 **1.1 回合内连打引擎（PlayCount Engine）** 与 **1.2 新抽牌增幅引擎（FreshDraw Engine）**、**1.4 最新牌连击计数器与核心循环（Combo Streak Engine）**。

- **支线一：混色拿来主义（Non-Red First Cast Engine）**  
  - 通过“本回合首次打出非红牌 → 获得红牌专用资源与放大”来鼓励 **非红 → 红 → 非红 → 红……** 的出牌序列。  
  - 详细规则见 **1.5 混色拿来主义能量引擎**。

- **支线二：短期透支与燃尽（Burnout）**  
  - 少量牌允许你用下回合/下两回合虚弱，换取本回合额外能量/抽牌/爆发伤害。  
  - 详细规则见 **1.3 短期透支与燃尽（Burnout）**。

- **随机性：高波动的导入/导出变体**  
  - 随机抽/随机生成他色牌 → 对应主引擎与混色支线的“随机导入”；  
  - 随机目标/奖励 → 对应连打/连击/燃尽的“随机导出”。  
  - 详细说明见 **1.6 随机性作为高波动导入/导出的变体**。

#### 1.1 回合内连打引擎（PlayCount Engine）
- **核心变量**  
  - `cards_played_this_turn`：本回合你已打出的牌数量（包含所有颜色）。
- **基础规则**  
  - 每当你打出一张牌：  
    - `cards_played_this_turn += 1`  
    - 某些红色效果以此为基数结算，例如：  
      - 当本回合你已打出第 3 / 5 / 7 张牌时，触发额外伤害 / 抽牌 / 返能等效果。  
- **设计意图**  
  - 奖励**高行动密度**：打得越多，本回合红色越强。  
  - 鼓励“拿来主义”：非红色的牌也能“喂养”红色引擎，让多色构筑与别色辅助牌变得更有价值。  
- **数值方向**  
  - 常见阈值：第 3、5 张牌作为标准触发点，第 7 张以上作为高稀有或爆发点。  
  - 护栏：单回合内基于 `cards_played_this_turn` 的收益应有上限，避免无限连打失控。

#### 1.2 新抽牌增幅引擎（FreshDraw Engine）
- **核心状态**  
  - “本回合新抽的牌”：在【本回合被抽上手】的牌（包含起手摸牌 + 中途抽牌）。  
  - 一张牌仅在被抽上的那个回合被视为“新鲜牌（Fresh）”，回合结束后失去该标记。
- **基础规则**  
  - 当你打出一张“本回合新抽的牌”时，若满足条件（如为红色或带特定标签），则获得额外收益，例如：  
    - 额外伤害、临时降费、返还部分能量、追加一次小效果等。  
- **设计意图**  
  - 奖励“**抽上来就用，不囤货**”的玩法，强化红色的冲动与即时兑现感。  
  - 把“随机抽到什么”转化为“只要立刻打出去就赚”的爽点，降低随机性的挫败感。  
  - 与“拿来主义”联动：只要是本回合新抽的牌，不论颜色，都可以被红色高效利用。  
- **护栏方向**  
  - “新鲜牌”加成只在抽上的那个回合生效；  
  - 较大的奖励（返能/额外抽牌）应限制为“每回合仅前 N 张新鲜牌触发一次”。

#### 1.3 短期透支与燃尽（Burnout）
- **核心变量**  
  - `red_overdrive_level`：你在本回合触发的“透支级爆发”的强度或次数（可用简单层数表示）。  
- **基础规则（由少量关键牌承载）**  
  - 当你在本回合通过红色牌获得**额外能量 / 大量额外抽牌 / 特大伤害爆发**时，会为接下来的 1–2 回合叠加“燃尽（Burnout）”效果，例如：  
    - 下回合起始能量 -1；  
    - 下回合少抽 1 张牌；  
    - 下回合第一张你打出的牌费用 +1。  
- **设计意图**  
  - 把“能量获得很危险”具体化为**短期节奏代价**，而不是长期诅咒。  
  - 区分红色与黑色：  
    - 红色：倾向“这一回合爆完，接下来一两回合很虚”，是冲动与宿醉。  
    - 黑色（未来）：可以承担“长期契约/永久代价”类玩法。  
- **护栏方向**  
  - 燃尽持续时间控制在 1–2 回合，不引入永久减益。  
  - 单场战斗内燃尽层数应有限制（例如最多 2 层），避免堆叠到不可挽回的状态。

#### 1.4 最新牌连击计数器与核心循环（Combo Streak Engine）
- **核心变量**  
  - `combo_streak`：本回合中，**连续打出的“本回合新抽的牌”数量**。  
- **基础规则**  
  - 当你打出一张本回合新抽的牌时：  
    - `combo_streak += 1`。  
  - 当你打出一张**不是本回合新抽的牌**，或回合结束时：  
    - `combo_streak = 0`。  
- **核心循环（流派一 · 最新牌连打流）**  
  - **核心能力牌 A（引擎）**：  
    - 规则口径：**“当你打出一张本回合新抽的牌时：抽 1 张牌。”**  
    - 作用：保证你持续获得“本回合新抽的牌”，支撑长连打。  
  - **核心攻击牌 B（连击奖励）**：  
    - 规则口径示例：  
      - “造成少量基础伤害，然后再造成等于当前 `combo_streak` 的伤害。”  
      - 或：“造成等于当前 `combo_streak` 的伤害（至少 1 点）。”  
    - 作用：奖励**连续使用最新牌**的操作，让玩家感到“越讲越上头、越打越狠”。  
- **设计意图**  
  - 将“本回合新抽的牌”从纯资源概念提升为**操作节奏轴**：  
    - 玩家通过“连续使用新抽牌”维持 `combo_streak`，  
    - 再用连击攻击牌或终结技，将该计数导出为一回合内的爆发。  
  - 挫败感控制：  
    - 连击牌本身应有**最低保底伤害**，即使 `combo_streak = 1` 或被打断，也不会完全变成废牌。  
  - 与 1.2 的关系：  
    - 1.2 定义“什么是本回合新抽的牌”，1.4 在此之上定义“**连续使用新抽牌**”的核心循环，是流派一的主玩法骨架。

#### 1.5 混色拿来主义能量引擎（Non-Red First Cast Engine）
- **能量体系背景**  
  - 本作能量分为两类：  
    - **有色能量**：由“理想”提供，每回合按颜色分配固定点数（如每个已解锁理想每回合 +1）。  
    - **万能能量**：每回合基础提供 3 点，可用于支付任意颜色费用。  
  - 非红色牌往往需要对应有色能量或万能能量来支付费用。  
- **核心规则（流派二 · 混色拿来主义）**  
  - 关键能力牌示意：  
    - “本回合你**首次**打出一张非红色牌时：获得 1 点**仅能用于打红色牌的万能能量**，并使你的下一张红色攻击牌伤害 +X 或费用 -1。”  
  - 设计要点：  
    - 奖励“每回合至少打一张非红牌”，但不强制连续多张非红。  
    - 通过“非红 → 红”的节奏，让玩家自然形成 **非红 → 红 → 非红 → 红……** 的有趣出牌序列。  
    - 使用“用途受限的万能能量”而非直接赠送有色能量，避免破坏其他颜色的资源曲线。  
  - 与 1.1 / 1.2 的关系：  
    - 非红色牌同样提升 `cards_played_this_turn`，并可能成为本回合新抽的牌；  
    - 该引擎让“打非红牌”既喂红色连打与最新牌循环，又提供一次性资源与伤害放大，是流派二的核心入口。

#### 1.6 随机性作为高波动导入/导出的变体（Randomized Import/Export）
- **定位**  
  - 随机性不作为第四套独立引擎存在，而是作为前述引擎（连打、新抽牌、混色能量）的**高波动变体**：  
    - 随机抽牌 / 随机生成他色牌 → 本质上是对 `FreshDraw` 与混色引擎的“随机导入”。  
    - 随机目标 / 随机奖励（伤害 / 抽牌 / 能量三选一等）→ 视为对现有计数器的“随机导出”。  
- **设计意图**  
  - 规则层：不再新增新的计数器或资源类型，避免红色被拆成过多互不关联的子系统。  
  - 体验层：玩家依然可以将“偏随机”的构筑视作红色的第三流派，但它在底层共享 `cards_played_this_turn`、`FreshCount/combo_streak` 与“首次非红”能量引擎。  
  - 结果：随机牌为前两大流派提供“高波动入口”和“赌博式爆发”，而不是重新发明一套完全独立的玩法。

### 2. 术语口径（卡面用语草案）

- **“本回合新抽的牌”**  
  - 定义：在本回合被抽上手的牌（包含起手 + 中途抽牌），直到回合结束。  
  - 卡面示例用语：  
    - “当你打出一张本回合新抽的牌时：……”  
    - “本回合新抽的牌费用 -1（仅本回合）。”  

- **“连打计数”**  
  - 定义：`cards_played_this_turn` 的玩家向表达。  
  - 卡面示例用语：  
    - “若这是你本回合打出的第 3 张牌：……”  
    - “当你本回合已打出 5 张或更多牌时：……”。  

- **“燃尽”**  
  - 定义：由红色透支带来的短期负面状态，仅影响接下来 1–2 回合。  
  - 卡面示例用语：  
    - “下回合开始时：你少 1 点能量（燃尽）。”  
    - “下 2 回合开始时：起手少抽 1 张牌（燃尽）。”

### 3. 红色性格与五维能力映射

- **输出：高**  
  - 核心通过“连打引擎 + 新抽牌增幅”制造回合内爆发和多段伤害。  
- **防御：低**  
  - 防御多为顺带或短期自保，不支持长期堆叠防御获胜。  
- **资源：中（短期爆发型）**  
  - 通过少量透支工具在关键回合爆发，不提供蓝色/绿色那种稳定、高效的资源滚雪球。  
- **牌流：中偏上（积极搜索）**  
  - 具备较高的过牌效率，但设计上要求“抽上来就用”，不鼓励长时间屯牌。  
- **控制：中（粗粒度控制）**  
  - 通过强迫敌人在不利时机行动、额外吃伤害、交出资源来施压，而不是精细队列排序或状态算计。

---

### 4. 卡牌分层与配比（红色轴心版本）

- **连打/爆发牌（约 40%）**  
  - 随本回合出牌量提升伤害或触发追加效果，是红色的主要输出来源。  

- **新抽牌增幅（约 30%）**  
  - 奖励“抽上来就用”的行为，为本回合新抽的牌提供降费、增伤或追加动作。  

- **透支/宿醉工具（约 15%）**  
  - 小部分关键牌允许你用下回合/下两回合虚弱作为代价，换取本回合的能量或牌差爆发。  

- **粗粒度控制与压制（约 10–15%）**  
  - 例如：迫使敌人立刻发动攻击、在不利时机结算技能、受到额外惩罚等。  

- **防御与自保（≤ 10–15%）**  
  - 小量自信/减伤往往与“已经高输出/多连打”联动，用于防止彻底玻璃炮，而不是独立防御轴。

### 5. 基础卡（建议首批最小可玩包 · 概念版）

> 仅描述功能与意图，不锁死数值，留给后续数值版本打磨。

1) **头脑一热（Skill，1）**  
- 效果：抽 2。  
- 若这两张牌在本回合被打出：每张牌额外造成少量伤害或返还少量能量。  
- 设计意图：定义“新抽就用”的基础爽感，鼓励玩家把抽牌与连打绑在一起。  

2) **一口气说完（Attack，1）**  
- 效果：对一个敌人造成中等伤害。  
- 若这是你本回合打出的第 3 张或之后的牌：改为造成显著更高的伤害。  
- 设计意图：最直接的“连打越多越疼”的标志性单体攻击。  

3) **借题发挥（Skill，1）**  
- 效果：本回合，当你打出非红色牌时：对随机敌人造成少量伤害。  
- 设计意图：将“拿来主义”具体化，把别色牌当作红色引擎的燃料。  

4) **预支口才（Skill，0–1）**  
- 效果：本回合获得额外能量；下回合开始：能量 -1，少抽 1 张牌（宿醉）。  
- 设计意图：定义红色“短期透支”的基线体验，代价看得见、可接受但明显。  

5) **舌灿莲花（Skill，1）**  
- 效果：本回合你每抽到第 2 张及之后的牌时：该牌本回合费用 -1（仅限本回合）。  
- 设计意图：把“积极搜索”与“当回合打光”强绑定，避免屯牌控库。  

6) **情绪外放（Attack，2）**  
- 效果：对所有敌人造成少量伤害。  
- 若本回合你已打出 ≥5 张牌：改为造成中等群体伤害，并对主目标额外造成一次单体伤害。  
- 设计意图：结合连打阈值的 AOE 爽点，强调“先铺手，再爆发”的节奏。  

7) **讲完再说（Skill，1）**  
- 效果：本回合结束时，若你本回合打出了 ≥X 张牌：立即抽牌，但下回合少抽若干张。  
- 设计意图：把“抽牌爆发”和“次回虚弱”绑定为一个清晰的决策点。

### 6. 数值与护栏（方向）

- **连打阈值**  
  - 常用阈值为第 3 / 5 张牌；第 7 张及以上适合用于高稀有或构筑向卡牌。  

- **新抽牌奖励**  
  - 对单张牌的加成不宜超过该牌基础价值的 50–70%，防止“全靠新抽牌”变成唯一最优策略。  
  - 大型奖励应限制触发次数（如“每回合至多触发 1–2 次”）。  

- **宿醉强度**  
  - 单次透支的典型代价：  
    - 下回合 -1 能量；或  
    - 下回合少抽 1 张牌。  
  - 稀有情况下可以叠加两种效果，但总层数应有限制（例如宿醉最多叠加 2 层）。  

- **防御上限**  
  - 红色很少拥有“纯防御牌”；典型防御效果绑定在“已经连打多张牌后的小幅自保”上，避免演化出独立防御轴。  

---

### 7. 与状态/资源的协同（草案）

- **与【破防】协同**  
  - 连打与多段伤害天然放大破防收益，红色适合承担“吃破防红利”的一方，而非主要施加者。  

- **与【规整 / 严谨 / 坚定】的区隔**  
  - 红色不负责稳定防御或精细资源管理，只在“已经高输出/多连打”时顺带获得一点保护或续航。  

- **与多色牌池的协同**  
  - 所有颜色的牌都能提升 `cards_played_this_turn`；  
  - 部分红色效果仅要求“打牌”，不要求“打红色牌”，鼓励多色构筑与拿来主义玩法。  

---

### 8. 遥测与验收指标（方向）

- **平均每回合出牌数**  
  - 红色应显著高于其他颜色（例如 1.2–1.5 倍），但不应频繁出现无限连打回合。  

- **透支使用率**  
  - 在中高难度战斗中，玩家平均每若干回合会选择至少一次透支工具；  
  - 过高代表代价太轻，过低代表体验太吓人，需要微调。  

- **宿醉期间的胜率与体感**  
  - 宿醉回合不应等同于“自动送命”，但应让玩家明显感到“节奏放慢、需要调整策略”。  

- **新抽牌利用率**  
  - “本回合新抽的牌被当回合打出的比例”应明显高于其他颜色，以验证“抽上来就用”的红色情绪是否成立。

### 9. 红色 v1 三大流派与 25 张卡牌配比（草案）

> 本节仅为“配额规划”，方便后续在具体设计时对号入座；数值与最终牌名留待 v2+ 打磨。

#### 9.1 三大流派概述

- **流派一：最新牌连打流（核心流）**  
  - 主题：围绕“本回合新抽的牌”展开，抽到就用，越拖越亏。  
  - 体验：节奏快、可控度高，是红色最稳定、最容易上手的玩法。  

- **流派二：混色拿来主义流（构筑流）**  
  - 主题：通过使用【非红色牌】与多种颜色的牌来喂养红色引擎。  
  - 体验：要求一定构筑理解度，上限高，适合喜欢多色套牌与联动的玩家。  

- **流派三：随机抽取与随机拿牌流（波动流）**  
  - 主题：随机抽、随机生成、随机用；高波动导入 + 高波动导出。  
  - 体验：围绕“看脸但爽”的情绪，适合作为娱乐向与高熟练度玩家的副流派。

#### 9.2 类型配比：Attack / Skill / Power

- **总量：25 张牌** （不含混色牌与未来扩展）  
- **按类型分布（建议）**：  
  - Attack：**10 张左右**  
    - 以导出伤害为主，部分兼具导入/倍增功能。  
  - Skill：**11 张左右**  
    - 负责过牌、资源操作、导入计数、混色拿来主义与软控制。  
  - Power：**4 张左右**  
    - 主要承担引擎与倍增效果（规则修改、被动触发），包括最新牌引擎与混色引擎。

#### 9.3 功能向配比：进攻 / 防御 / 引擎与功能

- **进攻向牌（约 14–16 张）**  
  - 主要为 Attack + 进攻型 Skill：  
    - 其中 6–8 张服务于“最新牌连打流”（按 PlayCount/FreshCount 放大伤害）。  
    - 4–5 张服务于“混色拿来主义流”（按非红色牌数量/不同颜色数量导出伤害）。  
    - 2–3 张为“随机流”准备（随机目标、随机数值的高波动导出）。  

- **防御与自保向牌（约 3–4 张）**  
  - 不构成独立防御流派，只作为高风险玩法的必要安全垫：  
    - 1–2 张“连打后顺手给自信/减伤”的防御 Skill。  
    - 1–2 张“当本回合计数器足够高时，换取一次中等防御或软控制”的工具。  
  - 原则：防御只救急，不提供“屯防御获胜”的路线。  

- **引擎 / 资源 / 功能向牌（约 5–7 张）**  
  - 以 Power 和部分 Skill 为主：  
    - 2–3 张支撑“最新牌连打流”的核心规则与放大（新抽→再抽，新抽→伤害）。  
    - 2–3 张支撑“混色拿来主义流”（打非红牌/多色牌获得资源或计数）。  
    - 1–2 张承担“短期透支与宿醉”的资源工具（事后代价换当回合爆发）。

#### 9.4 三大流派与配额对应（概览）

- **最新牌连打流（核心流 · 约 10–12 张）**  
  - Power：2–3 张（新抽→再抽、新抽→伤害、连打阈值奖励）。  
  - Attack：4–5 张（按 PlayCount/FreshCount 结算单体/群体伤害）。  
  - Skill：3–4 张（过牌、重抽、把旧牌变“新牌”的过滤工具）。  

- **混色拿来主义流（构筑流 · 约 7–8 张）**  
  - Power：1–2 张（打非红牌/不同颜色牌获得资源或计数）。  
  - Attack：2–3 张（按“本回合非红牌数/颜色数”导出爆发）。  
  - Skill：3–4 张（临时让非红牌视为新牌、多色触发奖励、构筑向调和牌）。  

- **随机抽取与随机拿牌流（波动流 · 约 4–5 张）**  
  - Attack：2–3 张（按计数对敌我随机目标导出多段伤害）。  
  - Skill：2 张左右（随机抽/随机生成他色牌，并与“新抽/混色”引擎兼容）。  
  - 备注：随机流不单独吃配额，会与前两流派共享部分导入/导出牌。
