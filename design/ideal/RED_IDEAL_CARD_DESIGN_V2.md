## 红色 · 影响轴 v2（设计空间与评分框架）

> 本文档在 v1 的基础上，收敛为**可计数、可评分、可生成卡牌的设计蓝图**。  
> 不直接落单卡，而是用「机制轴 × 功能轴 × 强度/权重」的模式来描述红色卡牌空间，方便后续按权重打分与挑选。

---

### 1. 版本目标与与 v1 的关系

- **目标**  
  - 将 v1 中分散的引擎描述（连打 / 最新牌 / 燃尽 / 混色 / 随机）整合为**一套主引擎 + 两条支线 + 随机变体**的清晰结构。  
  - 用**模式（Patterns）**代替单张卡牌，作为后续“按权重生成卡牌”的基础单元。  
  - 为后续的“打分与权重分配”预留明确的挂点（例如：输出 / 防御 / 资源 / 节奏 / 随机性等维度）。

- **与 v1 的关系**  
  - v1 保留为**完整语义版**（适合阅读理解与叙事）；  
  - v2 为**结构化版**（适合规划、计数与工具化），未来如有冲突，以 v2 为准进行实现层取舍。

---

### 2. 引擎总览（供模式引用）

> 本节只做摘要，详细语义仍参考 v1 第 1 章。

- **主引擎：最新牌连打循环（Fresh + Combo + PlayCount）**  
  - `cards_played_this_turn`：本回合已打出的牌数。  
  - Fresh：本回合新抽上手的牌。  
  - `combo_streak`：本回合连续打出的 Fresh 数量。  
  - 语义：**抽上来就用 → 连打 → 连击越高越疼**。

- **支线一：混色拿来主义（Non-Red First Cast）**  
  - 本回合首次打出非红牌 → 获得“仅能用于红牌”的万能能量 + 下一张红牌放大。  
  - 语义：**他色开关，红色吃饭**。

- **支线二：短期透支与燃尽（Burnout）**  
  - 当回合额外能量/抽牌/爆发 → 下一回合/下两回合能量与抽牌变差。  
  - 语义：**这一回合爽完，接下来几回合很虚**。

- **随机变体：随机导入 / 随机导出**  
  - 导入：随机抽牌/生成他色牌，但都视为 Fresh，可喂主引擎。  
  - 导出：用当前计数器（PlayCount / combo_streak / Burnout）对随机目标结算高波动效果。

### 3. 设计坐标系

#### 3.1 机制轴

- **新抽牌机制**：本回合新抽上手的牌，所有“最新牌(Fresh)”相关的触发与加成都挂在这里。  
- **回合出牌数机制**：依赖 `cards_played_this_turn`（本回合已打出牌数）的效果。  
- **连击次数机制**：依赖 `combo_streak`（连续使用新抽牌的次数）的效果。  
- **非红触发机制**：依赖“非红色牌”的使用，尤其是“本回合首次打出非红牌”的效果。  
- **燃尽机制**：涉及短期透支，下回合 -能量 / -抽牌 / +费用 等节奏代价。  
- **随机导入机制**：随机抽牌、随机生成他色牌，用来喂给最新牌与连打引擎。  
- **随机导出机制**：随机选择目标或随机选择奖励类型（三选一等），用当前计数器来结算高波动效果。

> 补充术语约定：**“创新牌” = 本回合新抽上手的牌**（即满足新抽牌机制定义的牌）。

> 后续模式中会用这些标签的组合来标记一张牌的机制特征。

#### 3.2 功能轴

- **单体伤害**：对一个目标造成伤害。  
- **群体伤害**：对多个目标或全体造成伤害（包含溅射、多段等）。  
- **防御**：获得自信 / 护甲 / 减伤 / 临时护盾。  
- **能量类资源**：能量相关操作（+能量、能量退款、费用转化等）。  
- **牌流类资源**：抽牌 / 过滤 / 换牌 / 轻度检索。  
- **控制与干扰**：打断行动、强制出牌、重排顺序、施加减益状态等。  
- **自我代价**：自伤 / 降低自己某项能力（不含燃尽，燃尽单独算在机制轴里）。  

#### 3.3 强度与权重轴

> 这一轴暂不写具体数值，由你后续提供权重并一起细化。这里只定义几个“档位”。

- **低基础**：基础数值明显低于同费基准，必须靠触发效果才划算。  
- **中基础**：基础数值接近同费基准，触发成功后略赚。  
- **高基础**：基础数值略高于同费基准，即便不触发也不亏。  
- **轻加成**：触发收益小（大约 +1 攻 / +1 护甲 / +1 抽 的级别）。  
- **中加成**：触发收益中等，玩家能明显感觉到“这下赚了”，但不至于爆表。  
- **重加成**：触发收益大，接近终结技级别，必须配合强护栏（例如触发条件苛刻、或必然附带燃尽代价等）。

### 4. 模式库 v2：红色卡牌设计模式

> 每个“模式”可以理解为一个“卡牌设计槽位”：  
> - 用三类标签来描述：用了哪些机制、承担什么功能、基础数值和加成大致处于哪个档位；  
> - 之后可以为每个模式分配权重（例如：红色在“新抽 + 单体伤害 + 中等基础 + 中等加成”上的总权重上限），再从中挑选要真正做成卡牌的模式。

#### 4.1 攻击向模式

- **A1：高基础 + 轻触发的单体打击**  
  - 机制轴：可以挂“新抽”或“回合出牌数”，但不是必须。  
  - 功能轴：单体伤害。  
  - 强度轴：高基础 + 轻加成。  
  - 语义：  
    - 作为红色的“主力平砍牌”之一，本身就不弱；  
    - 若目标牌是 Fresh 或在本回合第 3 张之后使用，则获得小幅额外伤害。  
  - 用途：  
    - 适合作为起始牌组升级版，或中罕见度的“稳定输出件”。

- **A2：低基础 + 强“最新牌连击”收尾**  
  - 机制轴：新抽 + 连击次数。  
  - 功能轴：单体伤害 / 多段伤害（按设计选择）。  
  - 强度轴：低基础 + 重加成。  
  - 语义：  
    - 基础伤害偏低，但若在高 `combo_streak` 时使用（且自己是 Fresh），可打出非常高的爆发。  
  - 护栏：  
    - 需求高连击或特定顺序；  
    - 不能轻易在任何回合直接打出“超额值”。  

- **A3：基于出牌数量的节奏爆发**  
  - 机制轴：回合出牌数。  
  - 功能轴：群体伤害，或“单体 + 溅射”。  
  - 强度轴：中基础 + 中加成。  
  - 语义：  
    - 在本回合前几张打出时只是普通打击；  
    - 在第 3/5 张之后打出时，启动“回合收尾爆发”的角色。  

#### 4.2 防御与自保模式

- **B1：弱基础 + 新抽触发的小防御**  
  - 机制轴：新抽。  
  - 功能轴：防御。  
  - 强度轴：低基础 + 中加成。  
  - 语义：  
    - 若这张牌或上张牌是 Fresh，则提供略高于基线的防御；  
    - 若触发失败，作为“勉强可用的弱防牌”。  

- **B2：连打后的小自保**  
  - 机制轴：回合出牌数。  
  - 功能轴：防御 +（可选）小抽牌 / 小返能量。  
  - 强度轴：中基础 + 轻加成。  
  - 语义：  
    - 当你本回合已经打出多张牌时，这张牌会给一点额外护甲/减压；  
    - 鼓励玩家将其安排在回合中后段，用它“收尾止血”。

#### 4.3 资源与燃尽模式

- **C1：强透支资源牌**  
  - 机制轴：燃尽 +（能量资源 或 牌流资源）。  
  - 功能轴：资源爆发。  
  - 强度轴：中基础 + 重加成（必须带燃尽）。  
  - 语义：  
    - 当回合获得显著能量/抽牌优势；  
    - 下回合必定遭受节奏惩罚（-能量 / -抽牌 / +费用），叠 1 层 `BURNOUT`。  

- **C2：轻量润滑资源牌（无燃尽）**  
  - 机制轴：回合出牌数 +（牌流资源 或 能量资源）。  
  - 功能轴：小抽牌 / 小返能量。  
  - 强度轴：中基础 + 轻加成。  
  - 语义：  
    - 不改变长期资源格局，只让连打更顺手；

#### 4.4 混色拿来主义模式

- **D1：首次非红 → 红色资源**  
  - 机制轴：非红触发 +（能量资源 或 单体伤害增益）。  
  - 功能轴：把他色牌转化为红色爆发燃料。  
  - 强度轴：中基础 + 中加成。  
  - 语义：  
    - 每回合首次打出非红牌时，获红色专用能量与下张红牌加成；  
    - 不叠燃尽（否则过于沉重），但强化“非红 → 红”的拆牌节奏。

- **D2：非红计数 → 一次性清算**  
  - 机制轴：非红触发 + 回合出牌数。  
  - 功能轴：单体伤害 或 牌流资源。  
  - 强度轴：低基础 + 中/重加成（需在牌面上显式写上限）。  
  - 语义：  
    - 按本回合用过的非红牌数量清算一次爆发或抽牌；  
    - 上限清晰，以免非红堆太多时失衡。

#### 4.5 随机高波动模式

- **E1：随机导入（随机新抽）**  
  - 机制轴：随机导入 + 新抽。  
  - 功能轴：牌流资源 / 伪检索。  
  - 强度轴：中基础 + 中加成（具体效果取决于随机池）。  
  - 语义：  
    - 看多几张牌或随机获得他色牌，全部视为 Fresh；  
    - 强化“这一回合看到啥就打啥”的即兴感。

- **E2：随机导出（随机目标爆发）**  
  - 机制轴：随机导出 +（回合出牌数 或 连击次数）。  
  - 功能轴：群体伤害 / 溅射伤害。  
  - 强度轴：低基础 + 重加成（通过高波动实现高上限）。  
  - 语义：  
    - 以当前计数器为基础，对随机敌人进行溅射或高波动爆发；  
    - 不做为核心稳定伤害来源，而是作为“赌一把”的娱乐向选择。

### 5. 权重与评分占位（等你给出权重后再一起填）

> 本节是后面“给红色在各个维度上分配总预算、然后给每个模式打分”的预留结构，现在先写框架。

- **5.1 红色在六个维度上的总权重（示例结构）**  
  - 输出（伤害）：`W_伤害`  
  - 防御：`W_防御`  
  - 资源 / 节奏：`W_资源`  
  - 牌流（抽牌 / 过滤）：`W_牌流`  
  - 控制 / 干扰：`W_控制`  
  - 随机性（波动感）：`W_随机`  

- **5.2 模式打分表（示意）**  
  - 每个模式 P（例如 A1、A2…）都会得到一个“在六个维度上的评分向量”：  
    - `Score(P) = (伤害, 防御, 资源, 牌流, 控制, 随机)`  
  - 总约束思路：  
    - ∑P Score(P) × Cards(P) ≈ 红色在五色坐标系中我们想要的“总权重”；  
    - 如果某个维度超标，就减少对应模式的牌张数；如果某个维度偏低，就补充相应模式。

> 等你先给出“红色在这六个维度上的目标权重”（比如：输出 5、防御 1、资源 3…）之后，我们就可以一起给上面的 A/B/C/D/E 这些模式打等级，再根据预算决定：v1 具体需要哪些模式、每种做几张牌。
