## 红色 · 影响轴 v2（设计空间与评分框架）

> 本文档在 v1 的基础上，收敛为**可计数、可评分、可生成卡牌的设计蓝图**。  
> 不直接落单卡，而是用「机制轴 × 功能轴 × 强度/权重」的模式来描述红色卡牌空间，方便后续按权重打分与挑选。

---

### 1. 版本目标与与 v1 的关系

- **目标**  
  - 将 v1 中分散的引擎描述（连打 / 最新牌 / 混色 / 随机）整合为**一套主引擎 + 一条支线 + 随机变体**的清晰结构。  
  - 用**模式（Patterns）**代替单张卡牌，作为后续“按权重生成卡牌”的基础单元。  
  - 为后续的“打分与权重分配”预留明确的挂点（例如：输出 / 防御 / 资源 / 节奏 / 随机性等维度）。

- **与 v1 的关系**  
  - v1 保留为**完整语义版**（适合阅读理解与叙事）；  
  - v2 为**结构化版**（适合规划、计数与工具化），未来如有冲突，以 v2 为准进行实现层取舍。

---

### 2. 引擎总览（供模式引用）

> 本节只做摘要，详细语义仍参考 v1 第 1 章。

- **主引擎：前沿触发循环（Fresh Trigger）**  
  - Fresh：本回合新抽上手的牌。  
  - 触发核心：凡“打出前沿牌”即可触发对应收益（增伤/回能/补牌/增严谨）。  
  - 语义：**抽上来就用 → 触发前沿 → 当回合持续滚雪球**。

- **支线一：混色拿来主义（Non-Red First Cast）**  
  - 本回合首次打出非红牌 → 获得“仅能用于红牌”的万能能量 + 下一张红牌放大。  
  - 语义：**他色开关，红色吃饭**。

- **随机变体：随机导入 / 随机导出**  
  - 导入：随机抽牌/生成他色牌，但都视为 Fresh，可喂主引擎。  
  - 导出：用当前计数器（前沿触发次数 / 非红触发次数）对随机目标结算高波动效果。

### 3. 设计坐标系

#### 3.1 机制轴

- **前沿牌机制**：以“单次抽牌动作”为批次，本次抽到的所有牌，统称为一批**前沿牌**；每一批中，只有**第一张打出的牌**可以吃到“前沿”加成，其余同批牌在该张牌结算后视为普通牌。  
- **连击次数机制（可选次级）**：依赖 `combo_streak`（连续使用新抽牌的次数）的效果；仅用于少量牌的“连接器”设计，不作为红色主轴。  
- **非红触发机制**：依赖“非红色牌”的使用，尤其是“本回合首次打出非红牌”的效果。  
- **随机导入机制**：随机抽牌、随机生成他色牌，用来喂给最新牌与连打引擎。  
- **随机导出机制**：随机选择目标或随机选择奖励类型（三选一等），用当前计数器来结算高波动效果。

> 补充术语约定：**“前沿牌” = 某次抽牌动作中新抽上手的那一批牌；该批次中，首张被打出的牌视为“消耗前沿”，其余同批牌在该张牌结算后不再享受前沿加成。**

> 后续模式中会用这些标签的组合来标记一张牌的机制特征。

#### 3.1.1 前沿触发设计守则（杀戮尖塔取向）

- **守则1：组合强于单卡**：强度来自“前沿触发器 + 前沿收益牌 + 收尾牌”的拼装，而非单卡超模。  
- **守则2：前沿次数是主计量**：对外可读指标优先使用“本回合前沿触发次数”，弱化显式连击计数。  
- **守则3：触发即时可感**：每次前沿触发都应有小反馈（数值跳字/短文案/音效），形成连续爽感。  
- **守则4：先设护栏再提强度**：前沿相关抽牌/回能/增伤都需要每回合触发上限，防止无限引擎。  
- **守则5：随机用于高光，不用于保底**：随机导出制造名场面，但不承担稳定胜率主线。

#### 3.2 功能轴

- **单体伤害**：对一个目标造成伤害。  
- **群体伤害**：对多个目标或全体造成伤害（包含溅射、多段等）。  
- **防御**：获得自信 / 护甲 / 减伤 / 临时护盾。  
- **能量类资源**：能量相关操作（+能量、能量退款、费用转化等）。  
- **牌流类资源**：抽牌 / 过滤 / 换牌 / 轻度检索。  
- **控制与干扰**：打断行动、强制出牌、重排顺序、施加减益状态等。  
- **自我代价**：自伤 / 降低自己某项能力（作为可选调味，不作为红色主轴）。  

#### 3.3 强度与权重轴

> 这一轴暂不写具体数值，由你后续提供权重并一起细化。这里只定义几个“档位”。

- **低基础**：基础数值明显低于同费基准，必须靠触发效果才划算。  
- **中基础**：基础数值接近同费基准，触发成功后略赚。  
- **高基础**：基础数值略高于同费基准，即便不触发也不亏。  
- **轻加成**：触发收益小（大约 +1 攻 / +1 护甲 / +1 抽 的级别）。  
- **中加成**：触发收益中等，玩家能明显感觉到“这下赚了”，但不至于爆表。  
- **重加成**：触发收益大，接近终结技级别，必须配合强护栏（例如触发条件苛刻、每回合次数上限等）。

### 4. 模式库 v2：红色卡牌设计模式

> 每个“模式”可以理解为一个“卡牌设计槽位”：  
> - 用三类标签来描述：用了哪些机制、承担什么功能、基础数值和加成大致处于哪个档位；  
> - 之后可以为每个模式分配权重（例如：红色在“新抽 + 单体伤害 + 中等基础 + 中等加成”上的总权重上限），再从中挑选要真正做成卡牌的模式。

#### 4.1 攻击向模式

- **A1：高基础 + 轻触发的单体打击**  
  - 机制轴：可以挂“新抽”或“连击次数”，但不是必须。  
  - 功能轴：单体伤害。  
  - 强度轴：高基础 + 轻加成。  
  - 语义：  
    - 作为红色的“主力平砍牌”之一，本身就不弱；  
    - 若目标牌是 Fresh 或在 `combo_streak` 达到阈值时使用，则获得小幅额外伤害。  
  - 用途：  
    - 适合作为起始牌组升级版，或中罕见度的“稳定输出件”。

- **A2：低基础 + 强“最新牌连击”收尾**  
  - 机制轴：新抽 + 连击次数。  
  - 功能轴：单体伤害 / 多段伤害（按设计选择）。  
  - 强度轴：低基础 + 重加成。  
  - 语义：  
    - 基础伤害偏低，但若在较高 `combo_streak` 时使用（且自己是 Fresh），可打出显著爆发。  
  - 护栏：  
    - 需求高连击或特定顺序；  
    - 不能仅靠单卡在任何回合直接打出“超额值”。

- **A3：前沿触发的节奏爆发**  
  - 机制轴：新抽（前沿）或连击次数。  
  - 功能轴：群体伤害，或“单体 + 溅射”。  
  - 强度轴：中基础 + 中加成。  
  - 语义：  
    - 未触发前沿时只是普通打击；  
    - 当本牌为前沿牌（或连击达阈值）时，启动“回合收尾爆发”的角色。  

#### 4.2 防御与自保模式

- **B1：弱基础 + 新抽触发的小防御**  
  - 机制轴：新抽。  
  - 功能轴：防御。  
  - 强度轴：低基础 + 中加成。  
  - 语义：  
    - 若这张牌或上张牌是 Fresh，则提供略高于基线的防御；  
    - 若触发失败，作为“勉强可用的弱防牌”。  

- **B2：连打后的小自保**  
  - 机制轴：连击次数。  
  - 功能轴：防御 +（可选）小抽牌 / 小返能量。  
  - 强度轴：中基础 + 轻加成。  
  - 语义：  
    - 当你本回合已经打出多张牌时，这张牌会给一点额外护甲/减压；  
    - 鼓励玩家将其安排在回合中后段，用它“收尾止血”。

#### 4.3 资源润滑模式（无延期惩罚）

- **C1：轻量润滑资源牌**  
  - 机制轴：连击次数 +（牌流资源 或 能量资源）。  
  - 功能轴：小抽牌 / 小返能量。  
  - 强度轴：中基础 + 轻加成。  
  - 语义：  
    - 不改变长期资源格局，只让连打更顺手；

- **C2：前沿触发型资源回补**  
  - 机制轴：新抽 +（能量资源 或 牌流资源）。  
  - 功能轴：当回合节奏回补。  
  - 强度轴：低/中基础 + 中加成。  
  - 语义：  
    - 在“抽上来就打”成功时返还能量或抽牌；  
    - 未触发时保留基础功能，避免手感过硬。

#### 4.4 混色拿来主义模式

- **D1：首次非红 → 红色资源**  
  - 机制轴：非红触发 +（能量资源 或 单体伤害增益）。  
  - 功能轴：把他色牌转化为红色爆发燃料。  
  - 强度轴：中基础 + 中加成。  
  - 语义：  
    - 每回合首次打出非红牌时，获红色专用能量与下张红牌加成；  
    - 强化“非红 → 红”的拆牌节奏。

- **D2：非红触发 → 红牌减费引爆**  
  - 机制轴：非红触发 + 红牌费用操控。  
  - 功能轴：节奏爆发 / 回合内手牌倾泻。  
  - 强度轴：低基础 + 中/重加成（需在牌面上显式写上限）。  
  - 语义：  
    - 你越积极使用非红牌，手中的红牌越便宜；  
    - 通过“减费→连打→收尾”的回合爆发获得收益，而不是单张清算。

#### 4.5 随机高波动模式

- **E1：随机导入（随机新抽）**  
  - 机制轴：随机导入 + 新抽。  
  - 功能轴：牌流资源 / 伪检索。  
  - 强度轴：中基础 + 中加成（具体效果取决于随机池）。  
  - 语义：  
    - 看多几张牌或随机获得他色牌，全部视为 Fresh；  
    - 强化“这一回合看到啥就打啥”的即兴感。

- **E2：随机导出（随机目标爆发）**  
  - 机制轴：随机导出 +（连击次数 或 非红触发次数）。  
  - 功能轴：群体伤害 / 溅射伤害。  
  - 强度轴：低基础 + 重加成（通过高波动实现高上限）。  
  - 语义：  
    - 以当前计数器为基础，对随机敌人进行溅射或高波动爆发；  
    - 不做为核心稳定伤害来源，而是作为“赌一把”的娱乐向选择。

### 4.y 前沿 × 小批注核心循环（讨论稿，优先实现）

> 目标：先做一套“只吃前沿”的红色爽感循环。  
> 思路参考小刀流，但不直接照搬数值：用 `小批注(0费, 4伤害, 消耗)` 作为“连段颗粒”，前沿负责点火与放大。

#### 4.y.1 循环骨架（四段式）

- **起手（点火）**：先用“抽牌/生成小批注”的牌启动手牌密度。  
- **放大（增压）**：用前沿触发牌把“小批注质量”抬起来（增伤/返能/补抽）。  
- **兑现（收尾）**：用一张中费攻击把当回合的前沿与小批注数量转成爆发。  
- **续航（不断流）**：用小防御+生成或抽牌，保证下一回合还能继续连段。

#### 4.y.2 先做的 6 个卡槽（仅前沿版本）

1) **前沿批注（Attack 1）**  
- 基础造成中等单体伤害；若为前沿牌，额外生成 1 张`小批注`。

2) **速记连舞（Skill 1）**  
- 生成 2 张`小批注`；若为前沿牌，再抽 1 张。

3) **前沿加注（Power 1）**  
- 本场战斗：你打出的`小批注`伤害 +X；若本牌为前沿牌打出，本回合额外 +1。  
- 备注：对应“小刀增伤器”的红色实现，但把爆发窗口绑定在前沿触发。

4) **灵感刀雨（Skill 2）**  
- 生成 3 张`小批注`；若为前沿牌，本回合你下一张攻击牌费用 -1（最低 0）。

5) **快评快打（Attack 1）**  
- 造成基础伤害；若为前沿牌，获得 1 点能量。  
- 备注：作为连段润滑件，保证“打小批注不亏节奏”。

6) **前沿清算（Attack 2）**  
- 基础伤害 + 额外伤害（按“本回合已打出的小批注数量”结算，建议设上限）。  
- 备注：这是“兑现件”，不应脱离前沿链条单卡超模。

#### 4.y.3 前沿五引擎牌（第一批核心）

> 说明：以下 5 个引擎牌用于统一“前沿触发循环”。  
> 优先级：先做 1/2/3，再做 4，最后做 5（风险最高）。

1) **三次前沿回能（Power）**  
- 规则：本战中，每触发 3 次前沿，获得 1 点能量。  
- 建议护栏：每回合最多触发 1 次。

2) **前沿增伤（Power）**  
- 规则：本战中，你打出的前沿牌伤害 +X。  
- 建议护栏：X 从小值起步（建议 1 或 2）。

3) **下一张视为前沿（Skill）**  
- 规则：本回合你下一张打出的牌视为前沿牌。  
- 作用：作为“润滑器”，降低前沿链断档概率。

4) **前沿得严谨（Power）**  
- 规则：本战中，每次触发前沿，获得 1 点严谨。  
- 建议护栏：改为“每 2 次前沿 +1 严谨”或“每回合前 2 次前沿才生效”。

5) **前沿抽牌（Power）**  
- 规则：本战中，每次触发前沿，抽 1。  
- 风险：极高，最容易形成无限。  
- 建议护栏：改为“每回合首次前沿抽 1”或“每回合最多触发 2 次”。

#### 4.y.4 设计护栏（避免单轴过强）

- **护栏A：小批注增伤要有软上限**（例如单回合最多结算到 +N）。  
- **护栏B：返能与补抽分离**（同一张牌尽量不要同时给“稳定返能 + 稳定抽牌”）。  
- **护栏C：清算牌写明上限**（例如最多按 4~5 张小批注计入加成）。  
- **护栏D：强牌依赖前沿触发**（脱离前沿时仍可用，但效率明显下降）。  
- **护栏E：前沿抽牌类优先做“每回合首次触发”版本**（作为默认安全模板）。

#### 4.y.5 体验目标（为什么先做这一组）

- 回合内手感：**快**（0费连续打牌）+ **响**（每张都有反馈）+ **爆**（末段清算）。  
- 构筑层手感：不是一张神卡，而是“起手件/放大件/兑现件”拼起来才强。  
- 与现有实现对齐：可优先复用 `frontier_bonus`、`create_card_in_hand`、`gain_energy`、`draw`，降低首轮实现成本。

### 5. 权重与评分占位（等你给出权重后再一起填）

> 本节是后面“给红色在各个维度上分配总预算、然后给每个模式打分”的预留结构，现在先写框架。

- **5.1 红色在六个维度上的总权重（示例结构）**  
  - 输出（伤害）：`W_伤害`  
  - 防御：`W_防御`  
  - 资源 / 节奏：`W_资源`  
  - 牌流（抽牌 / 过滤）：`W_牌流`  
  - 控制 / 干扰：`W_控制`  
  - 随机性（波动感）：`W_随机`  

- **5.2 模式打分表（示意）**  
  - 每个模式 P（例如 A1、A2…）都会得到一个“在六个维度上的评分向量”：  
    - `Score(P) = (伤害, 防御, 资源, 牌流, 控制, 随机)`  
  - 总约束思路：  
    - ∑P Score(P) × Cards(P) ≈ 红色在五色坐标系中我们想要的“总权重”；  
    - 如果某个维度超标，就减少对应模式的牌张数；如果某个维度偏低，就补充相应模式。

> 等你先给出“红色在这六个维度上的目标权重”（比如：输出 5、防御 1、资源 3…）之后，我们就可以一起给上面的 A/B/C/D/E 这些模式打等级，再根据预算决定：v1 具体需要哪些模式、每种做几张牌。
