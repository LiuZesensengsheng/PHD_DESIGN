## 敌人设计维度总览（Enemy Design Axes v1）

> 目的：把“这个敌人到底从哪些角度被设计出来的？”说清楚，方便以后选敌人、做表格、给别人解释。  
> 本文只讲 **标签/维度**，不讲具体数值和技能（那些在 `TEMPLATE_ENEMY_PROTO` / `INTENT_SPEC` / `INTENTS_CATALOG` 里）。

---

## 1. 世界观 / 剧情向维度

- **场景（scene / slice）**  
  - 敌人出现在哪一类生活场景：组会、宿舍、实验室、财务处、论文系统、亲戚饭局……
  - 用于：做章节分布、关卡分布、事件联动。

- **谁在伤害我（who_hurts_me）**  
  - `SELF`：自己（拖延、自我怀疑、完美主义…）  
  - `PEOPLE`：导师、同学、甲方、家人、学生等“具体人”  
  - `SYSTEM`：制度、流程、评审机制、报销/投稿系统  
  - `WORLD`：时代、运气、随机灾难（疫情、裁员、方向凉了）
  - 用于：做敌人生态统计、平衡“自虐 vs 外部压迫”的配比。

- **现实梗来源（source）**  
  - 来自哪条 life event / 真实槽点，便于追溯和筛选（例如 `life_events::ddl_yesterday`）。

- **发疯度（madness_level）**  
  - `0 = 现实`：几乎原样搬运现实；  
  - `1 = 夸张`：把现实放大到好笑的程度；  
  - `2 = 超现实 / Meta`：心魔具象化、Excel 地狱、系统拟人等。  
  - 用于：控制整局/整 Act 的“疯度曲线”，避免全程都太沉重或太抽象。

---

## 2. 情绪功能维度（emotion_function）

> 敌人主要是用来让玩家 **扎心？爽？反思？** —— 一个敌人可以有 1–2 个主标签。

- `扎心`：  
  - 让玩家觉得“操，这也太真实了”；更多是情绪共鸣与自嘲。
- `爽`：  
  - 给玩家一个“老子终于反杀/报仇”的爽点；通常是“行为很恶心但本体脆皮/奖励高”。
- `反思`：  
  - 让玩家稍微停一下，想想“自己是不是也这样”；多用在 SELF 敌人或系统级敌人上。

用途：
- 控制一条路线或一场战斗里三种情绪的比例；
- 筛选 Demo / 宣发用的代表敌人（例如：首发多选“扎心+爽”，少量“反思”）。

---

## 3. 颜色 / 理念维度（color_hint & ideals）

- **颜色提示（color_hint）**  
  - 使用五色体系：`WHITE / BLUE / BLACK / RED / GREEN` 或双色组合。  
  - 粗略含义：  
    - WHITE：秩序 / 规则 / 程序正义  
    - BLUE：知识 / 理性 / 信息优势  
    - BLACK：野心 / 利己 / 代价换收益  
    - RED：输出 / 冲动 / 影响力  
    - GREEN：厚积 / 拖延 / 长期收益

- **理念冲突 / Hook**  
  - 这个敌人天然和哪些理想冲突、诱导哪些妥协（用于心魔/审判联动）。  

用途：
- 作为敌人“颜色身份”的补充（和 `ACT1_ROSTER` / 各色敌人文档呼应）；  
- 在未来版本中，与理想系统、心魔卡、Excel Hell 等做挂钩。

---

## 4. 战斗 / 数值向维度

- **角色定位（role）**  
  - 示例标签：  
    - `PRESSURE_DPS`：压力输出主力  
    - `STALLER`：拖延/拉长战斗  
    - `RESOURCE_DRAINER`：掏空能量/抽牌/金币/灵感  
    - `CURSE_POLLUER`：往牌库塞废牌/心魔  
    - `SUPPORT / SUMMONER`：给队友加盾/召唤增援  
  - 用于：快速理解这只敌人在一场战斗中的“功能位”。

- **战斗钩子（combat_hook）**  
  - 一句话解释“为什么这只敌人有记忆点”：  
    - 例如：  
      - “倒计时敌人：每回合减少倒计时，归零时大爆发”；  
      - “强迫你连续打同费/同色，否则施压并加废牌”；  
      - “疯狂重排你手牌，把你上回合的努力全部抹掉”。

- **意图 / 技能组合（intents & skills）**  
  - 基于 `INTENTS_CATALOG` 选 2–4 个 Intent，配：  
    - 冷却、权重、条件（见 `INTENT_SPEC`）；  
    - 与上面的 `role` / `combat_hook` 一致。

- **数值带（bands）**  
  - HP 区间、单次施压区间、诅咒/状态叠加速度等，按：  
    - 普通 / 精英 / Boss 不同档位取不同 band。  

用途：
- 方便程序接入时，一眼知道这只敌人需要哪些意图/参数；  
- 在平衡时，可以按 role 纵向对比不同敌人的数值带。

---

## 5. 实际落地建议

- 对于每一个“正式收录”的敌人原型，建议在设计表/CSV 中至少包含：  
  - `enemy_key`  
  - `display_name_zh`  
  - `scene`  
  - `who_hurts_me`  
  - `emotion_function`  
  - `madness_level`  
  - `color_hint`  
  - `role`  
  - `short_flavor`（1 句话段子）  
  - `combat_hook`（1 句话机制钩子）

- 实现上可以：  
  - 在 `enemies_life_events_v1_index.csv` 基础上继续扩列这些字段；  
  - 或者在敌人 JSON / 原型表中增加对应字段，用于编辑器筛选和统计。
