# 论文线 Boss 2.0 · 期刊投稿

> 版本口径：概念设计 2.0，仅描述体验与机制结构，不牵涉实现细节与具体数值。
> 核心改动：  
> - 敌方从“主编 + 三审稿人小怪”重构为“期刊系统本体 + 三颗审稿充能球”  
> - 己方从“单人作战”改为“博士本人 + 论文作为队友单位”  
> - 三阶段职能重排：  
>   - 阶段1：论文第一次站出去挨打（初审/门槛）  
>   - 阶段2：正式审稿战（三颗球 + 换位）  
>   - 阶段3：最后搓出巨大的 response letter（投稿质量权重），真正的结果推迟到战役层若干回合后揭晓

---

## 0. 叙事与情绪目标

- 还原的不是一场“普通 Boss 战”，而是：  
  - 你和自己的论文，作为一个共犯组合，一起被期刊系统审判；  
  - 你可以选择替论文挡刀，或者让论文替你挨打；  
  - 在系统眼里，你们只是编号与勾选项，但对你来说，这是几年生命的缩影。
- 关键情绪：
  - **共同受难**：你和论文都在掉血，看谁先撑不住。  
  - **责任转移与内疚**：你到底愿不愿为论文扛下那些刀？  
  - **最后一搏**：在有限回合里把能写的回复都写完，明知道结果未必好。  
  - **时间冻结与延迟宣判**：打完 Boss 很久之后，某一天才突然弹出那封邮件。

## 0.1 2026.01.08 确定事项（口径锁定）

> 这是一份“设计决议记录”，用于统一后续文档与实现口径。

- **1) 灵感掉落**：确定大小战斗都会掉落「灵感」，灵感初始携带**正面/负面标签**（混合存在）。
- **2) 战后标签编辑**：确定战斗结束后的奖励阶段可以对标签进行**去除或添加**（以 n 选 1/多选一方式呈现；未来可做 CoC 风格叙事包装）。
- **3) 负面标签功能（MVP：L1 低耦合）**：确定负面标签会让论文 Boss **变强**；首版 Boss 强度只读取**负面数量**（不逐条耦合），后续可扩展到类别/严重度。
- **4) 正面标签语义**：确定正面标签代表论文“血量/生命线”（投稿资格的直观表达），可选附带少量战斗能力。

## 1. 登场与整体结构

- 敌方：`期刊系统本体 + 三颗审稿充能球`
  - 本体代表“编辑部 + 主编的最终意志”，有耐久度（或进度条），不直接大量输出。
  - 三颗球代表三类审稿口味/规则轴：方法 / 理论 / 结果导向。
- 己方：`博士本人 + 论文队友`
  - 博士：有生命值（精神/身体）、压力条、理想系统。  
  - 论文：以**正面标签作为“血量/生命线”**（投稿资格的直观表达），并附带少量抽象属性（创新性/严谨度/语言合规等）。
  - 特殊机制：**换位牌**（Placeholder 名称），允许你在“自己与论文之间”切换受击目标或承担代价。

- 三个阶段：
  1. **阶段1：初审与第一次露面** —— 论文第一次站出去挨打，定下期刊口味与偏见。  
  2. **阶段2：正式审稿战（三球 + 换位核心玩法）** —— 主要战斗、节奏与资源博弈发生在这里。  
  3. **阶段3：最后的 response letter** —— 在有限回合中强化论文与回复；战斗结束后进入战役等待期，真正结果延迟揭晓。

## 2. 己方：论文队友与换位机制

### 2.1 论文队友（Paper Ally）

- **论文“血量/生命线”由正面标签表达（无传统HP条）**
  - 论文不采用传统意义上的 HP 数值条；它的“还能不能算一篇成型论文”，由两组信息共同表示：  
    - **存活的正向标签**：尤其是 2–3 个核心卖点标签（核心结论、方法亮点等）；  
    - **累积的黑话标签**：审稿意见中已经被写进评语的各种槽点。

- **论文标签环（核心设计）**
  - 论文以一个角色立绘/图标呈现，周围环绕一圈可见的「正向标签」：  
    - **来源**：  
      - 定方向 / Idea 阶段：产出 2–3 个「核心创新点/卖点标签」（如“方法新颖”“问题重要”“跨学科”）。  
      - 实验 / 写作阶段：叠加若干「方法严谨」「样本扎实」「语言漂亮」「图表合格」等执行层标签。  
      - 特殊事件：可能追加「业界合作」「基金加持」「大数据集」等 bonus 标签。
    - **表现**：  
      - 正向标签：亮色、小徽章样式，悬停可显示一行描述。  
  - 与此同时，单独维护一组「黑话标签」（负向）：  
    - 仅由审稿球/系统的攻击或前序偷懒行为生成，如「立意不明确」「贡献不清晰」「结果夸大」等；  
    - 不要求与某个正向标签一一对应，只在数量与类型层面参与结算。

- **论文属性（抽象数值层）**
  - 在标签之上，保留少量抽象属性作为数值总览（用于简化计算）：  
    - 创新性（低/中/高）  
    - 方法严谨度（低/中/高）  
    - 语言合规 / 图表合规（低/中/高）  
  - 这些属性可以视为所有相关标签的「合成向量」，用于：  
    - 决定某些审稿球的基础行为与倾向；  
    - 为阶段3的 response 生成提供输入特征。

- **标签受击（简化规则）**
  - 精准打击：某些攻击不会打你，而是指定**移除 1 个相关正向标签**（从论文标签环上摘掉一个）：  
    - 如：方法球优先移除「方法」类标签；绿色球优先移除「基础/文献」类标签等。  
  - 黑话生成：部分攻击会**直接追加 1 条黑话标签**到列表中，而不是从正向翻面。  
  - 纯视觉剥落：在移除标签或新增黑话时，可以用标签破碎、掉落等视觉反馈，形成“简历被一条条划掉”的感受。

- **存活条件与投稿资格（概念口径）**
  - 论文被视为“仍可投稿”的最低条件可以设为：  
    - 核心卖点标签中至少保留 1 个未被移除；  
    - 黑话标签数量低于某个阈值（否则视为本刊无缘）。  
  - 当上述条件破裂时：  
    - 直接进入「本轮无缘发表」的失败分支，或触发极端代价选项（牺牲理想/长期负面遗物）来“强行维持这篇论文的生命线”。

- **论文作为“半被动单位”**
  - 不频繁主动出牌，但可以具有若干被动/触发效果：  
    - 如：拥有某些特定正向标签时，被对应球攻击会反弹少量“兴趣”→ 接受倾向；  
    - 语言/图表合规高时，可自动抵消一部分格式类打击或缩短等待时间。  
  - 在阶段3的 response 中，**剩余的正向标签集合与黑话标签列表**将作为生成选项与文案的主要输入，驱动非战斗叙事流程。

### 2.2 换位牌与站位（Bodyguard / Swap）

- **站位规则（简化版）**
  - 战场前排只容纳 1 个单位：要么是你本人，要么是论文；另一位自动在后排。  
  - 敌人的大多数攻击默认只命中**前排单位**；少量技能为 AOE，会同时命中前后排。  
  - 某些特殊攻击标记为“修改负担（穿透）”，这类攻击无视护甲：  
    - 若命中你 → 转化为压力/精力消耗（你真的去改了）；  
    - 若命中论文 → 生成 1 条黑话标签（你没改，让稿子背锅）。

- **换位牌（每回合一次简单决策）**
  - 每回合你通常会获得 1 张简单的「换位」牌：  
    - 使用后，你与论文立即交换前后排位置；  
    - 不改变敌人本回合的意图类型，只改变这次攻击更可能打在谁身上。  
  - 这张牌不引入额外资源条，仅用于表达：  
    - “这一次，我替论文挡刀”；  
    - 或“这一次，让论文自己去挨那下合理的批评”。

- **情绪与长期后果（概念口径）**
  - 频繁让论文挨打：  
    - 更多正向标签会被移除 / 更多黑话累积 → 更难投稿成功；  
    - 久而久之，你对“作品”的情感连接会削弱，偏向 Excel 化。  
  - 频繁替论文挨打：  
    - 自己压力与理想裂痕剧增，易触发 SAN 事件；  
    - 但会强化某些理想路线（“为作品负责”），在后续剧情中被记住。

## 3. 敌方：期刊系统与三颗审稿充能球

### 3.1 期刊系统本体

- 表面形态：后台界面 + 主编头像/抽象化“审判席”。  
- 数值上：
  - 有一条“系统耐久度”或“审稿流程进度条”，代表整套机械还能运转多久。  
  - 被你成功抵消/驳回打击、完成修改时，这条条会下降，推动战斗阶段转换。
- 行为：
  - 少量直接攻击，多数时候以“全局指令”方式影响三颗球与你们：  
    - 如“本回合所有审稿球行动两次”、“下回合玩家能量上限-1”、“清除所有你的正面状态”等。

### 3.2 五色理想审判球（Ideal Orbs）

> 球 = 一种理想颜色在期刊审判下的化身。  
> 每颗球拥有：常驻被动 + 充能规则 + 充满时的大招 + 对对应理想/论文标签的长期影响。

- **白色理想之球（秩序 / 合规 Orb）**  
  - 含义：流程、规范、伦理、格式、制度性要求。  
  - 常驻被动：  
    - 根据其充能层数，增加各种「形式/流程」类打击：如每层使未满足格式要求的卡牌额外+1 能量，或让“未按指引操作”的行为产生额外压力。  
  - 充能规则：  
    - 当你多次无视格式/伦理/流程提示（例如不处理 Desk 给的提醒、反复用“抄近路”牌）时，为其加层。  
  - 充满大招（“全面合规审查”）：  
    - 对所有与「语言/图表/伦理声明」相关的论文标签进行一次检定：  
      - 合规高 → 保留甚至强化相关标签，缩短等待时间，小幅提升接受倾向；  
      - 合规低 → 摧毁或黑化若干标签（如“图表合格”→“图表潦草”），并对论文完整度造成打击。

- **蓝色理想之球（知识 / 方法 Orb）**  
  - 含义：方法严谨、可复现性、统计/推理的扎实度。  
  - 常驻被动：  
    - 每层“方法质疑”会削弱基础质量收益，或让某些关键牌（方法类）效果打折。  
  - 充能规则：  
    - 当你使用明显偷懒或不严谨的方案（如“凑样本”“随便拟合”类牌），或在高压力下反复重复同一套路时加层。  
  - 充满大招（“方法总审查”）：  
    - 对「方法相关标签」进行一次大检定：  
      - 成功：生成/强化“方法严谨”类标签，获得「理论深度/方法强化」被动，并提升接受倾向；  
      - 失败：摧毁/黑化方法标签（“方法新颖”→“方法存疑”），产生长期心魔（“我是不是在自欺欺人”），并打击论文完整度。

- **绿色理想之球（深积累 Orb）**  
  - 含义：基础性、文献综述、长期积累与扎实程度。  
  - 常驻被动：  
    - 每层使你在未进行足够铺垫时打出的“高难度论断/结论牌”效果折扣，或额外+压力。  
  - 充能规则：  
    - 当你跳过文献/基础铺垫，直接打“大结论”“宏大叙事”时，为其加层。  
  - 充满大招（“你是否读过前人工作？”）：  
    - 询问论文上的「文献/基础标签」：  
      - 如果此前在论文线上认真经营过这类标签（综述战打得好），则获得“基础扎实”被动，减轻未来审稿打击；  
      - 若相关标签稀少，则标签被黑化为“重复工作”“没看文献”，并在阶段3 response 中增加额外解释负担。

- **红色理想之球（影响 / 戏剧性 Orb）**  
  - 含义：故事性、冲击力、社会/媒体影响，标题党的诱惑。  
  - 常驻被动：  
    - 每层提升对“效果巨大”“震撼性结论”类牌的诱惑（数值加成），但也放大被反噬的后果（被揭穿时伤害更大）。  
  - 充能规则：  
    - 当你使用过于夸张的叙事/标题、或频繁追求“爽点”而忽略严谨时，为其加层。  
  - 充满大招（“这真的有这么重要吗？”）：  
    - 对「影响/卖点标签」进行评估：  
      - 若确有扎实基础支撑（绿/蓝属性良好），则封为“高影响力工作”，大幅提升接受倾向，并生成强力正面遗物；  
      - 若基础薄弱，则标签被黑化为“过度宣传”，对你造成巨大压力，并在后续游戏中留下“自我怀疑/名不副实”的心魔。

- **黑色理想之球（野心 / 代价 Orb）**  
  - 含义：指标崇拜、p-hacking、过度包装、为了发表可以牺牲什么。  
  - 常驻被动：  
    - 每层降低你在“牺牲理想/Excel 化决策”时的心理门槛（相关牌更便宜、更强），同时在背后积累看不见的代价。  
  - 充能规则：  
    - 当你主动选择“抹平异常”“删掉不利数据”“为了过线牺牲理想”时，为其加层。  
  - 充满大招（“你愿意付出什么代价？”）：  
    - 引导一次关键抉择：  
      - 你可以牺牲一个理想、接受永久负面遗物或心魔，以换取论文属性/接受倾向的巨大跃升；  
      - 也可以拒绝交易，保持理想但承受短期数值上的不利（如这次投稿更可能失败）。  
    - 无论选择如何，都会在理想/SAN 轨迹上留下深刻印记。

- **球的激活数量与难度/期刊档位**
  - 为避免 5 球同时在线导致过载，每场 Boss 战仅激活 1–3 颗球：  
    - 低难度 / 普通期刊：激活 1–2 颗球（通常是论文主色 + 副色对应的理想）；  
    - 中等难度：激活 3 颗球（主色 + 副色 + 与其张力最大的对立色）；  
    - 顶刊 / 特殊挑战（T0/T1）：可解锁“多球审判模式”（例如 4 球 + 特殊 Excel 球），作为高难度可选战。  
  - 未被激活的理想球可通过：  
    - 阶段3 response 的文案与选项；  
    - 战役层终裁事件与长期心魔/美德，  
    - 以叙事方式出场，而不参与本场战斗的实时博弈。

## 4. 阶段1：初审与第一次露面（门槛战）

### 4.1 目标与定位

- 不是完整 Boss 阶段，而是：  
  - 带论文第一次进入系统视野；  
  - 锁定本次投稿的“期刊口味与偏见”；  
  - 给论文和你身上打上若干长效标签（影响接下来两阶段的起点）。

### 4.2 体验与演出

- 场景：投稿系统界面，主编不直接出现，只有“系统提示”和 Desk 检查列表。  
- 敌方形态：1–2 个“格式/范围问题”单位，或仅以状态条形式呈现。  
- 己方：你 + 论文共同上场，论文血条首次出现。
  - 口径修正（与 2026.01.08 决议一致）：此处“血条”指**正面标签生命线的可视化UI**，不是传统 HP 数值条。

### 4.3 机制要点（示意）

- **期刊选择（可选）**  
  - 若本次是主动冲高（T0/T1），可在此阶段通过小事件选择“稳妥 / 较野 / 魔法冷门”三类期刊口味。  
  - 决定三颗球的基础配置与阶段3门槛（稳妥期刊要求低但奖励一般，顶刊要求高且波动大）。

- **初始印象标签**  
  - 系统读取你整条论文线已有的风格标签（创新性/语言合规/图表合规…），在 UI 一侧以“Desk 印象”形式展示。  
  - 这些标签映射为：  
    - 某些球的初始充能层数；  
    - 论文初始状态（附带 1–2 个正负面 buff）；  
    - 审稿等待时间的基准值等。

- **小测试：格式/基本门槛**  
  - 1–2 回合内，系统抛出简单的格式/范围问题：  
    - 如“参考文献格式不符”、“图表分辨率不足”等。  
  - 你可以用低成本“修改”卡修补；  
  - 若应对得好：  
    - 为论文增加 1 个小的正向执行层标签（如“格式合规”）；  
    - 初始接受倾向+10 或略微缩短后续等待时间；  
  - 若应对糟糕：  
    - 在论文上生成 1 条轻微黑话标签（如“格式潦草”）；  
    - 某颗球初始充能+1；  
    - 甚至延长后续审稿等待时间。

## 5. 阶段2：正式审稿战（理想球 + 换位核心）

> 这是本 Boss 的主菜阶段：  
> - 动作密度高、决策压力大；  
> - 主要消耗你的卡组资源、精神状态与论文标签“血量”；  
> - 换位牌的价值在此阶段被完全放大。

### 5.1 基本循环

- 每回合流程示意：
  1. 期刊系统发布当回合指令（可能修改球行为/行动顺序）；  
  2. 已激活的理想球按各自规则尝试充能或施加被动效果；  
  3. 你选择出牌：  
     - 进攻系统耐久度；  
     - 针对某颗球“对症”减层或改变其下次大招效果；  
     - 使用换位牌决定本回合谁来承担打击；  
     - 使用防御/自疗/压压力牌维持自己和论文的状态。  
  4. 回合结束结算：压力、心魔计数、基础质量累积等。

### 5.2 论文死亡与特殊结局

- 若**论文在标签意义上“死亡”**（例如所有核心卖点标签被移除，或黑话标签数达到致命阈值）：
  - 立即进入“论文死亡”演出：
    - 系统提示：“Based on the current state, the manuscript cannot be considered for publication.”  
    - 你可以被给予一个残酷选择：
      - 直接本轮战斗失败 → 当前投稿梯子此级别期刊永久锁死；  

- 若**你本人先倒（精神崩溃）**：  
  - 触发理想系统中的特殊线（待后续细化），可能直接退出本年/本局。  
  - 论文可能以“孤儿稿”的形式被展示在某些后续事件中（彩蛋）。

### 5.3 推进到阶段3的条件

- 满足以下任一即可进入阶段3：
  - 系统耐久度下降到阈值以下；  
  - 所有已激活理想球都至少完成过一次大招循环；  
  - 论文仍保留至少 1 个核心卖点标签（即在“标签意义上”尚未死亡，仍有资格继续玩 response）。

---

## 6. 阶段3：最后的 response letter（战斗内收尾）

### 6.1 目标与体验

- 这一阶段不再强调“挨打”，而是强调：  
  - 你和论文坐下来，把所有审稿意见逐条整理、回应；  
  - 在有限的几个回合中，把能写的回复都写进去；  
  - 然后按下“Submit”，你们暂时分离。

- 体验关键词：  
  - 梭哈、专注、时间拉长、心里发虚但仍然在写。

### 6.2 场景与界面

- 战斗 UI 转换为“回复信编辑界面”：  
  - 左侧：审稿意见列表（来自三颗球的历史行为浓缩成若干条目）。  
  - 右侧：你的回复区域/论文结构概览。  
  - 下方：论文当前状态（完整度、关键属性）与你的压力/理想状况。

### 6.3 机制骨架（示意）

- 给予玩家 2–4 个“编辑回合”：
  - 每回合你可以：  
    - 使用特定“修改/回复”卡牌对应若干条审稿意见；  
    - 把资源投入到强化某个论文属性（如方法/理论/图表展示）；  
    - 选择是否为了适应期刊口味而牺牲理想（换取一段时间内回复效果翻倍）。  
  - 未被覆盖/回应的意见，将在结果生成时作为负权重参与计算。

- 回合结束时：
  - 生成一个隐藏的“投稿质量评分/权重”：  
    - 由论文当前状态 + 覆盖的意见比例 + 你的牺牲/坚持情况共同决定；  
  - 该评分不直接给出具体结果，只影响后续战役层的“Accept/Major/Reject 概率与分支”。

### 6.4 提交与离场

- 最后一回合，你按下“Submit response”：  
  - 论文队友的模型/血条慢慢淡出，仅在战役 UI 中保留一个“Submitted Manuscript”图标或遗物；  
  - 本场 Boss 战斗结束，回到战役甘特界面。  
  - 当前年/回合继续推进，你在其他轨道上过日子、被别的事折磨。

### 6.5 设计取向补充（白描 + 留白）

- 机制侧的核心简化口径：  
  - 对玩家来说，阶段3 只需要记住一件事：**在有限的几个关键选择中，尽量保护和放大论文剩余的核心创新点/标签**；  
  - 理想球在此阶段不再以数值输出为主，而是通过简短台词与被打掉/黑化的标签，提供审判氛围与风味。  
- 表现侧的目标：  
  - 用「白描 + 留白」的方式，仅通过幸存/损毁标签、极少量选项与简短台词，勾勒出整段研究生涯的缩影；  
  - 不试图在此处解释全部系统，只给足线索，让考究玩家在细节里读出完整故事。
战斗结束后应该给一些奖励。

## 7. 战役层：延迟结果与终裁事件

> 真正的“Accept / Major / Reject”不在 Boss 战内给出，而是作为战役层的后续事件。

### 7.1 等待期

- Boss 战结束后，在甘特上生成一个`审稿等待`任务块：  
  - length ≥ 2 回合（例如 ≥ 20 天）；  
  - 期间不发生与该 Boss 直接相关的战斗，只以泡泡/事件轻描淡写：  
    - “系统状态更新：With editor.”  
    - “Reviewer invited.”  
    - “No decision has been made yet.” 等。

- 等待期的体验：  
  - 你在做别的任务线（助教、项目、生活琐事），但脑子里总挂着这篇稿子；  
  - 偶尔弹出的泡泡会轻微影响压力或给极小的调整（如等待-1 天）。

### 7.2 结果事件

- 若干回合后，在战役层触发 `审稿结果` 事件：  
  - 根据隐藏的投稿质量评分 + 难度档位 + 少量随机，给出：  
    - Accept / Major Revision / Reject 等结果。  
  - 事件内再承接：  
    - 黑色幽默演出（邮件、印章等）；  
    - 你如何解释这次结果（自责/合理化/带伤前行）；  
    - 对理想/SAN 的长期影响与遗物/心魔发放。

## 8. 总结与后续细化点

- 本 2.0 方案的核心变化：
  - 把 Boss 战变成“博士本人 + 论文队友”对抗“期刊系统 + 1–3 颗五色理想审判球”的三阶段结构；  
  - 把阶段1–2 聚焦于“共同挨打 + 保护论文核心标签 + 换位决策”，阶段3聚焦于“基于幸存标签，在有限交互中搓出一封巨大的 response letter”；  
  - 把真正的审稿结果移到战役层的延迟事件中，强化“等待与未知”的味道。

- 后续可细化内容（留给未来版本）：  
  - 换位牌的具体牌面与升级分支；  
  - 理想球之间的联动招式（例如某两球共同充满时触发特殊连招）；  
  - 论文死亡/你本人崩溃时的特殊演出与分支路线；  
  - 终裁事件与理想/SAN 系统的接口（如何在结果阶段生成心魔或美德）。

## 9. MVP 与信息控制原则（实现护栏）

- **9.1 首战简化版**
  - 玩家第一次遇到论文 Boss 时：  
    - 仅激活 1–2 颗理想球（通常是论文主色 + 副色）；  
    - 论文身上仅展示 6–8 个「关键标签」，其余以抽象属性呈现；  
    - 阶段3 只包含 3–4 个关键交互，不出现复杂树状对话或大量数值结算。  
  - 高难度/后续周目可逐步解锁：  
    - 第 3 颗理想球；  
    - 更丰富的标签组合与 response 文案。

- **9.2 球与标签的复杂度上限**
  - 每颗理想球的设计控制为「1 个常驻被动 + 1 条充能条件 + 1 个大招效果」，避免过度堆叠规则。  
  - 每场战斗中，论文仅展示有限数量的可见标签（建议 6–8 个），并区分：  
    - 核心卖点标签（被打掉可能导致立即丧失投稿资格）；  
    - 一般执行层标签（被打掉主要影响数值与 response 文案）。

- **9.3 阶段3 的定位**
  - 明确为**非战斗叙事段落**：  
    - 不再有普攻与复杂出牌，仅通过少量选项来表达态度与取舍；  
    - 依赖 “白描 + 留白” 的方式，通过标签残骸与简短台词勾勒故事，而不是试图解释一切。  
  - 机制上保证：  
    - 选择对结果有**有限但真实的影响**（例如改变 10–20% 的结果倾向或理想变化），  
    - 但不会完全推翻阶段1–2 的战斗表现，避免让前面体验显得无意义。
