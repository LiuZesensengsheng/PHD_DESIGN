## 甘特图：任务与事件规范（Tasks vs Events）

目标：把“必须做的进度块”（任务）与“随机发生的叠加段子/微互动”（事件）解耦，既保证主线推进，也容纳黑色幽默内容而不打断节奏。

### 一、定义
- 任务（Task Block）
  - 必须处理，会消耗一个“事件轮”（进入一个战斗/遭遇/交互流程）。
  - 进度相关：影响路线进度、敌人数量、奖励、融合等。
  - 形态：战斗、遭遇、Boss、提交/审核等主流程节点。

- 事件（Event Overlay）
  - 随机触发，作为“图标”叠加在甘特块上方；可点击弹出文本/二选一；不进入战斗。
  - 不影响进度推荐与节点融合；默认不改变敌人数/奖励，仅做轻量叙事与小幅临时效果。
  - 用途：黑色幽默段子、世界观填充、微型抉择（可给少量自信/临时层/洞见等）。

### 二、调度与寿命
- 生成：每回合从事件池抽取 0–2 个“信封事件”，挂载到“当前/相邻”任务块上方。
  - 权重：ANY（全局）≈70%｜THEME（随当前主题偏置）≈20%｜LOCAL（与该块强相关）≈10%。
  - 去重冷却：同一事件ID在 5 回合内不重复。
  - 降权：Boss/关键战时仅投放“静默段子”或显著降权，避免出戏。
- 寿命（TTL）：默认存活 1–2 回合；未点击则过期并写入“邮件/消息流”。
- 限制：同一甘特块头部最多 2 个事件图标；事件不参与融合/波次。

### 三、UI 表现
- 任务：常规甘特块；点击进入（消耗一轮）。
- 事件：块上方小图标（可高亮/脉冲）；Hover 显示标题；点击弹窗文本/选项；过期则在侧边“事件Feed”记录。

### 四、与融合的关系（参见 GANTT_FUSION_RULES）
- 只有任务参与“数量融合+主题继承”。
- 事件不改变敌人数/奖励/主题；融合后仍按原主题的事件池生成新事件。

### 五、数据口径（最小原型）
```yaml
# 任务节点
task:
  id: paper_submission_01
  theme: FINALS           # 或 LAB / ROLL_CALL / PAPER
  type: TASK
  enemy_count: 3          # 用于融合
  has_boss: false
  duration_turns: 1       # 进入即消耗一轮

# 事件节点（叠加，不入融合）
event:
  id: meme_email_tpl_01
  theme: PAPER
  type: EVENT
  icon: envelope
  title: 模板拒稿：感谢您的宝贵工作
  body: 经过认真评审，我们认为您的研究非常有潜力，但不适合本刊。
  choices:
    - text: 算了，右移到附录
      effect: { gain_confidence: 3 }
    - text: 立刻改图1
      effect: { add_insight: 1 }
  ttl_turns: 2
```

### 六、事件池（示例，黑色幽默口径）
- 已读不回（icon: bell）→ 你+自信3；同时生成一条“群发邮件”日志
- 审稿人神评论（icon: comment）→ 抽取“洞见”1；下一次定焦收益+1
- Overleaf 合并冲突（icon: code-branch）→ 你随机弃1，抽1；显示“手动解决冲突”

### 七、实现提示（非强制）
- 抽象事件为“轻交互弹窗”，效果仅限小幅临时数值或日志；不修改路线图结构。
- 任务驱动主线；事件驱动叙事密度与基调，不增加系统复杂度。
- 模式开关：纯段子 / 轻微效果 / 关闭；按需在设置中切换。
