### 五色能力权重 v1（草案）

> 目的：给五色卡牌一个**统一的数值坐标系**，尤其是把「资源/节奏（轴 3）」当成整个系统的核心权重，用它来平衡其它四个维度。

---

### 1. 五个能力维度（轴）

- **轴 1：输出（Damage / 压力输出）**  
  - 单回合爆发、持续输出、点杀能力。
- **轴 2：防御（Block / 自保 / 风险控制）**  
  - 格挡、自信、减伤、拖延时间的能力。
- **轴 3：资源 / 节奏（最重要）**  
  - 能量获取与变形、费用操作、节奏控制（跳回合、额外行动、未来换现在等）。  
  - **说明**：这是整个系统的“货币层”，谁掌握资源，谁决定游戏节奏；数值上必须非常克制。
- **轴 4：牌流（抽牌 / 选牌 / 过滤）**  
  - 抽牌、检索、底牌调度、弃牌换牌等。
- **轴 5：控制 / 顺序（Control / Ordering）**  
  - 位移、打断、眩晕、锁队列、按自定义顺序结算等。

> 约束：**任何颜色如果在轴 3 上拿到“高”，其它轴必须对应削弱**；反之，如果资源能力弱，可以在输出/防御/控制上稍微放手一点。

---

### 2. 白色（秩序学派 / Pointer-Order）的权重定位 v1

> 参考 `IDEAL_SYSTEM_DESIGN_v9.0` 中的 **白色·秩序学派**：范式、标准、秩序治理。  
> 关键词：Pointer / Sorting / Queue / Audit  
> 主题：通过“排序 + 指针 + 规则”来管理风险与爆发，**更像一个调度者而不是纯输出机器**。

- **轴 1：输出 → 中偏高**  
  - 主要依赖：  
    - 指针小段伤害（PD）、位置优势（端点爆发）、多段结算。  
  - 爆发存在，但通常需要**事先布位 / 排队**，而不是裸给大数字。

- **轴 2：防御 → 中**  
  - 目标：整体防御能力中等，但**防御决策质量高**：  
    - 通过队列重排、指针安全线来把不可控伤害变成“能接得住的形状”；  
    - 防御牌常带有“顺便调整位置/顺序”的副作用。

- **轴 3：资源 / 节奏 → 低~中（强约束）**  
  - 原则：白色**不应该**是“能量发动机色”：  
    - 很少有额外能量；  
    - 费用折扣也应当稀少且带明确条件（如：仅对 Pointer/Sorting，或仅对防御牌）。  
  - 资源优势更多体现在：  
    - **用排序和节奏把有限资源用在“最关键的一列/一回合”**，而不是绝对能量变多。

- **轴 4：牌流 → 低**  
  - 几乎不靠爆抽获胜：  
    - 偶尔有“结题报告”式的结算牌，但多半是**按统计量结算**（这回合做了多少次移动/防御），而不是+3、+4 抽牌。  
  - 这迫使玩家更重视**出牌顺序**而不是“多摸几张再说”。

- **轴 5：控制 / 顺序 → 高（招牌）**  
  - 这是白色的绝对主轴：  
    - 敌人队列排序、端点/中位操作、指针定位、安全线、防御顺序调整。  
  - 很多牌都是“输出/防御 + 顺序副效果”：  
    - 例如“略微减伤 + 把高伤敌人排到后面”、“获得自信 + 把指针移到高威胁那路”等。

- **基础攻防基线 v1（用于起始基础牌）**  
  - 攻击：**6**  
  - 防御：**5**  
  - 说明：输出中偏高、防御中等，通过指针/端点/路径和顺序控制放大实际伤害与减伤。

### 3. 其它颜色（与 IDEAL SYSTEM 对齐的权重草案）

> 以下颜色描述参考 `IDEAL_SYSTEM_DESIGN_v9.0` 的五色思想轮盘，只定**相对权重与设计方向**，具体牌面以后在各自专门文档中细化。

- **蓝色（理论学派 · 真理色）**  
  - 输出：中  
    - 精准爆发、条件连锁，但通常需要前置准备（信息、标记）。  
  - 防御：中  
    - 通过“预知/减弱/延后结算”来减少实损，而不是堆巨额护甲。  
  - 资源：中~高（**偏高但必须受约束**）  
    - 费用变形、未来换现在、对抽牌堆/弃牌堆的“时间套利”。  
    - 若这里拉到“高”，必须压低蓝色在输出或防御上的上限。  
  - 牌流：高  
    - 蓝色是**抽牌/过滤的主力色**：看更多牌、筛选更好牌、洗掉坏牌。  
  - 控制：中~高  
    - 偏“信息控制型”：窥视、改写意图、推迟或重定敌方行动，而不是物理位移。

- **基础攻防基线 v1**  
  - 攻击：**4**  
  - 防御：**6**  
  - 说明：裸数值偏防御，输出依赖于牌流、资源与条件式爆发，体现“靠聪明与信息优势取胜”。

- **黑色（野心学派 · 权力色）**  
  - 输出：中~高  
    - 延迟爆发、多段 DoT、自伤换高伤，常见“伤害放大器”。  
  - 防御：低  
    - 更倾向“把损失转成资源/优势”，而不是真的防住伤害。  
  - 资源：高  
    - 主打“代价换资源”：扣血、弃牌、加压力，换能量/抽牌/临时强力效果。  
    - 这是黑色的招牌轴之一；对应代价必须显性且尖锐。  
  - 牌流：中  
    - 较多“择牌/弃牌”与墓地互动，但不会像蓝色那样疯狂抽牌。  
  - 控制：中~高  
    - 诅咒、削弱、压迫（压力）、偷取资源、强迫抉择等。

- **基础攻防基线 v1**  
  - 攻击：**6**  
  - 防御：**4**  
  - 说明：进攻中偏高、防御偏低，靠“自伤/代价换资源与放大”来在低防御前提下拉高整体强度。

- **红色（经世学派 · 影响色）**  
  - 输出：高  
    - 爆发、AOE、多段伤害、临时增强，是**最直接的伤害色**之一。  
  - 防御：低  
    - 防御多为“顺带”或短期自保，而不是长期堆叠。  
  - 资源：中  
    - 短期爆发（本回合额外能量/额外行动）、事后代价（下回合疲劳/减资源）。  
  - 牌流：中  
    - 通过“打更多牌”或“连锁打出”间接增加牌流，而不是蓝色式高效抽牌。  
  - 控制：中  
    - 偏“粗粒度控制”：硬性打断、强迫打牌、用高压迫使玩家交资源，而不是精细排序。

- **基础攻防基线 v1**  
  - 攻击：**7**  
  - 防御：**4**  
  - 说明：基础攻击最高、防御较低，后续通过连打、最新牌与短期“燃尽”进一步放大回合内爆发。

- **绿色（厚积学派 · 节律色）**  
  - 输出：中  
    - 较少爆发，更多是“每回合稳定输出”，靠时间滚雪球。  
  - 防御：高  
    - 护甲/自信/上限管理、恢复、减伤、缓慢净化负面效果——**最厚的长线防御色**。  
  - 资源：高  
    - 核心在于“结转与复利”：未用能量/护盾可以部分结转为下回合优势。  
  - 牌流：中  
    - 偏向“优化牌库质量”（移除废牌/压缩牌库）而非爆抽。  
  - 控制：中  
    - 调整自己行动节奏：“早收工换下回合红利”、“固定每回合操作上限但提高效率”。

- **基础攻防基线 v1**  
  - 攻击：**3**  
  - 防御：**7**  
  - 说明：基础输出最低、防御最高，主打护甲/上限/恢复与资源复利，靠时间滚雪球而非短期爆发。

> 备注：以上权重刻意让 **蓝/黑/绿** 在资源轴（3）上占据不同风格的“优势位”，  
> 白/红在资源轴上保持中~低，通过**顺序和爆发**而不是资源数量来参与对局节奏。这样更贴合 `IDEAL_SYSTEM` 对五色人格与理想冲突的描写。

### 4. 白色后续设计检查清单（面向实现的自检问题）

> 每次新增白色卡牌/机制时，用下面几条做快速自检：

- **资源轴（3）是否超线？**  
  - 这张牌是否在“白色以外的牌”也会显得过强的资源能力？  
  - 如果是，那应该属于蓝色/黑色，而不是白色。

- **是否自然地强化了轴 5（顺序/控制）？**  
  - 如果只加了伤害或格挡，没有任何顺序/位置/指针相关的副作用，是否应该换成别的颜色？  

- **防御是“规划型”而不是“单纯更厚”吗？**  
  - 这张防御牌是否在帮助玩家**重排/选择要防的伤害**，而不是单纯堆数字？

- **是否在牺牲抽牌（轴 4）的前提下，给了足够的“操作空间爽感”？**  
  - 换句话说：不靠抽更多牌，靠更聪明地用现有牌。

这一份文档只锁定“方向与权重”，具体每张牌/机制的设计仍以各色的详细设计文档（如 `WHITE_IDEAL_CARD_DESIGN_V3.md`）为准。今后如果某个颜色的实践明显偏离了这里的权重，我们再回到这里做 v2 调整。
