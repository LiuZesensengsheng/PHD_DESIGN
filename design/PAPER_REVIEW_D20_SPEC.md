# 论文线审稿 D20 方案草案（讨论稿）

> 目标：保留“狗屎运名场面”的传播乐趣，同时让论文质量与构筑仍然有意义。  
> 说明：本文件仅记录讨论结论，不包含代码实现。

---

## 1. 流程（当前共识）

1. 先选论文难度/投稿档位（T0~T4）。
2. 每次投稿前必须进行一次“写作投稿战”。
3. 写作投稿战胜利后执行投稿（0 回合动作）。
4. 投稿后进入等待期：
   - 基础等待：2~3 回合；
   - 随机事件可增加等待；
   - 总等待上限：6 回合。
5. 等待结束后进入审稿检定（D20）。
6. 若投稿失败：回到“写作投稿战”，不追加额外工作量阶段。
7. 每条论文线每回合最多投稿一次。

---

## 2. 检定核心公式

`审稿总值 = 1d20 + 论文质量修正 + 审稿人修正 + 其他修正`

与期刊阈值（DC）比较：

- 顶刊：`DC 25`
- 普刊：`DC 15`
- 垃圾刊：`DC 10`
- 水刊：`DC 5`

---

## 3. 结果分段（待概率回填）

> 说明：本段阈值先作为占位，后续按实际概率表回填与微调。

- `总值 >= 接收阈值`：直接接收（Accept）
- `总值 >= 小修阈值`：小修（Minor）
- `总值 >= 大修阈值`：大修（Major，可接审稿战）
- `总值 < 大修阈值`：拒稿（Reject）

---

## 4. 狗屎运原则（已确认）

1. 允许“低质量稿靠高点数中过高档期刊”的荒诞结果。
2. 该奇迹不附加惩罚，不追溯代价（纯奖励事件）。
3. 设计目标是“可赌赢”，并保留传播性名场面。
4. 同时允许反向极端：高质量论文也可能因恶心审稿人/坏运气而拒稿。

---

## 5. 优势/劣势机制（总条目 <= 6）

规则：

- 优势投：投 `2d20` 取高。
- 劣势投：投 `2d20` 取低。
- 同次检定若同时拥有优势与劣势，则互相抵消为普通投。

来源清单（6 条）：

1. `导师操刀`（Buff）：本次审稿检定获得优势。
2. `某种心流`（Buff）：本次审稿检定获得优势。
3. `邀稿`（Buff）：本次审稿检定获得正向修正（具体数值待定）。
4. `审稿人重派`（Debuff）：等待 `+1` 且本次检定劣势。
5. `格式事故`（Debuff）：本次检定负向修正（具体数值待定）。
6. `待策划补充`（TBD）：保留一个来源位，后续由策划统一定义。

注：第 6 条为预留槽位，不阻塞当前系统实现与联调。

---

## 6. 审稿人“老虎机”演出（可选层）

1. 编辑先抽出 3 名审稿人（如严苛/良心/轮空/摆烂）。
2. 每个审稿人给一个固定修正（例如 `-2 / +1 / 0 / 波动`）。
3. 修正并入 D20 总值，不单独替代最终结算。

用途：强化戏剧感与黑色幽默，不改变核心判定公式。

---

## 7. 调参锚点（便于后续数值迭代）

1. `论文质量修正`建议区间：`-3 ~ +6`
2. `审稿人总修正`建议区间：`-4 ~ +4`
3. 等待事件增量建议：每次 `+1`，总上限保持 `6` 回合
4. 先锁定 DC，不频繁改动；优先调“修正来源”与“优势/劣势频率”
